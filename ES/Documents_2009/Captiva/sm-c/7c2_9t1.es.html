DTC B1000 -

Fallo de funcionamiento de la ECU

descripción del circuito

La detección de fallos internos se gestiona dentro del módulo de control. No hay implicados circuitos externos.

Condiciones para la ejecución del DTC

El módulo ejecuta el programa para detectar un fallo interno al ordenarse la puesta en marcha. Lo único que hace falta es tensión y masa. Este programa se ejecuta aunque la tensión esté fuera del intervalo operativo válido.

Condiciones para la activación del DTC

El módulo detecte un fallo de funcionamiento interno.

Acción realizada cuando se activa el DTC

El módulo rechaza todas las entradas de datos adicionales.

Condiciones para borrar el DTC

El DTC se borra al desaparecer el fallo de funcionamiento.
El DTC histórico se borra cuando el contador de ciclos de encendido del módulo llega al umbral de puesta a cero de 50 ciclos sin que se haya repetido el fallo de funcionamiento.

Verificación del sistema/circuito

Si este DTC se recupera como un DTC actual, sustituya el módulo que lo activa.

INSTRUCCIONES DE REPARACIÓN

Lleve a cabo la Verificación de la reparación del diagnóstico una vez completada la reparación. Consulte la Sección 9A, Sistema de cableado de la carrocería. Consulte también el módulo relacionado y la sustitución, configuración y programación del módulo que no se esté comunicando.

DTC B3105 -

Llaveros del sistema del mando a distancia de las puertas no programados

descripción del circuito

El DTC B3105 se activa si el receptor de cierre a distancia de las puertas (RCDLR) no tiene memorizado ningún llavero.

Condiciones para la ejecución del DTC

El sistema de accionamiento de la función del mando a distancia (RFA) debe estar activado, efectuándose una prueba cada vez que la llave cambie de ciclo.

Condiciones para la activación del DTC

El código de diagnóstico se activa si no hay llaveros memorizados en el RCDLR.

Acción realizada cuando se activa el DTC

El sistema RFA no funciona.

Condiciones para borrar el DTC

El código se borra y deja de estar vigente cuando se programa al menos un llavero para el RCDLR.

INSTRUCCIONES DE REPARACIÓN

Lleve a cabo la Verificación de la reparación del diagnóstico una vez completado el procedimiento de diagnóstico. Consulte la Sección 9A, Sistema de cableado de la carrocería.

DTC B0155 -

Pérdida de comunicación con el módulo de control del cuadro del panel de instrumentos (IPC)

descripción del circuito

Los módulos conectados a los circuitos de datos serie GMLAN de alta y baja velocidad controlan las comunicaciones de datos serie durante el funcionamiento normal del vehículo. Los módulos se intercambian la información y las órdenes operativas. Los módulos tienen información registrada previamente sobre los mensajes que necesitan intercambiarse por los circuitos de datos serie, para cada red virtual. Los mensajes se supervisan y el módulo receptor también usa algunos mensajes periódicos para conocer la disponibilidad del módulo transmisor. Cada mensaje contiene el número de identificación del módulo transmisor.
El módulo de control de la carrocería (BCM) activa el circuito que habilita la comunicación de datos serie y el circuito de datos serie que reactiva los accesorios aplicando tensión cuando la llave de contacto está en la posición ACC, ON o START.

Condiciones para la ejecución del DTC

La tensión del sistema esté entre 9 y 16 voltios.
La matriz de modos de alimentación del vehículo necesite comunicación de datos serie para tener lugar.

Condiciones para la activación del DTC

No se haya recibido un mensaje periódico supervisado que incluya la disponibilidad del módulo transmisor.

Acción realizada cuando se activa el DTC

Nunca se envía señal de ningún módulo, por lo tanto, no funcionará ningún subsistema específico.

Condiciones para borrar el DTC

El DTC se borra al desaparecer el fallo de funcionamiento.
El DTC histórico se borra cuando el contador de ciclos de encendido del módulo llega al umbral de puesta a cero de 50 ciclos sin que se haya repetido el fallo de funcionamiento.

Ayudas para el diagnóstico

Utilice la lista de Descriptores del DTC anterior para determinar el módulo que no se está comunicando.
Si hay varios módulos sin comunicación, elija el más cercano al conector de diagnóstico (DLC).
Algunos módulos pueden no tener protección interna para salidas de tensión específicas y pueden interrumpir la tensión positiva de la batería o el fusible. Si un fusible de entrada de tensión está abierto y no se detecta ningún cortocircuito en ese circuito, asegúrese de que ningún módulo tenga un circuito de tensión de salida de derivado a masa antes de sustituir el módulo.
Este diagnóstico puede emplearse para cualquier módulo que se comunique con datos serie GMLAN de alta o baja velocidad diciendo que el equipo está equipado con la opción que usa dicho módulo.

INSTRUCCIONES DE REPARACIÓN

Lleve a cabo la Verificación de la reparación del diagnóstico una vez completado el procedimiento de diagnóstico. Sección 9A, Sistema de conexiones de la carrocería Consulte también el módulo relacionado y la sustitución, configuración y programación del módulo que no se esté comunicando.

DESCRIPCIÓN GENERAL Y FUNCIONAMIENTO DEL SISTEMA

Programación de transmisores

Importante: Todos los transmisores que deban ser reconocidos por el receptor de cierre a distancia de las puertas (RCDLR) deben programarse en una sola secuencia. Si se pone el sistema en modo de programación, se borrarán todos los transmisores programados con anterioridad al recibir la señal de programación del primer transmisor. Pueden programarse hasta 4 transmisores. No utilice ni programe los transmisores cerca de otros vehículos que estén en modo de programación del mando a distancia. Así se evitará programar los transmisores del vehículo equivocado.

  1. Instale un scanner.
  2. En el menú principal del scanner, seleccione el sistema de programación de servicio y siga las instrucciones en pantalla.

Importante: El scanner debe permanecer conectado hasta que se programen todos los transmisores.

  1. Mantenga pulsados al mismo tiempo el botón de cierre o bloqueo (LOCK) y el botón de apertura o desbloqueo (UNLOCK) de un solo transmisor. Transcurridos 5 segundos, el scanner indicará que el transmisor está programado.
  2. Repita el paso anterior para programar un máximo de 4 transmisores.
  3. Retire el scanner.
  4. Pruebe las funciones del transmisor para verificar que funcionan correctamente.

Sistema del mando a distancia de las puertas

El sistema del mando a distancia de las puertas es un dispositivo complementario para abrir y cerrar el vehículo. Use el sistema del mando a distancia de las puertas junto a una llave de cerradura. Las radiofrecuencias o pilas agotadas pueden inhabilitar el sistema.
El mando a distancia de las puertas permite accionar los siguientes componentes:
  • Las cerraduras de las puertas, incluida la del maletero
  • Las luces de entrada iluminada
  • La alarma de pánico
El sistema del mando a distancia de las puertas cuenta con los siguientes componentes principales:
  • Los transmisores
  • El receptor de cierre a distancia de las puertas (RCDLR). El RCDLR se comunica con el módulo de control de la carrocería (BCM) a través del circuito de datos serie del mando a distancia de las puertas. Indicación de fallo o alarma

El sistema del mando a distancia de las puertas emplea una tecnología de código alternante. La tecnología de código alternante impide que alguien pueda grabar el mensaje enviado desde el transmisor y lo utilice para acceder al vehículo. El término de "código alternante" se refiere a la forma en que el sistema del mando a distancia de las puertas envía y recibe las señales. El transmisor manda la señal cada vez en diferente orden. El transmisor y el RCDLR están sincronizados para usar el orden apropiado. Si un transmisor programado envía una señal en un orden que no espera el RCDLR, entonces el transmisor está desincronizado. Esto sucede tras pulsar 256 veces cualquier botón del transmisor estando fuera del alcance del vehículo.
