Descripción del circuito/sistema

El circuito de datos de serie es el medio con el que los módulos de control del vehículo se comunican entre sí. Una vez que la herramienta de diagnóstico se conecte al circuito de datos de serie a través del DLC, la herramienta de diagnóstico se puede usar para controlar cada módulo para hacer un diagnóstico y para comprobar los códigos de diagnóstico de avería (DTC). Cuando el interruptor de encendido está en MARCHA, cada módulo de control que se comunica a través del circuito de datos de serie envía un mensaje de estado de salud (SOH) para garantizar que el módulo de control está funcionando adecuadamente. Cuando el módulo de control deja de comunicarse a través del circuito de datos de serie, por ejemplo si el módulo pierde potencia o toma de tierra, desaparecerá el mensaje SOH que envía normalmente a través del circuito de datos de serie. Otros módulos de control en el circuito de datos de serie que esperan recibir ese mensaje SOH detectan su ausencia; a su vez, esos módulos de control establecen un DTC asociado con la pérdida de SOH del módulo de control que no se comunica.

Condiciones para la activación de DTC

La tensión del sistema está entre 9-16 V.

Condiciones para ajustar el DTC

No se ha recibido un mensaje supervisado periódicamente que incluye la disponibilidad del módulo transmisor.

Acción realizada cuando el DTC se ajusta

Subsistemas específicos no funcionarán.

Condiciones para el borrar el DTC
    • Un DTC actual se borra cuando ya no existe la avería.
    • Un DTC del historial se borra cuando el contador del ciclo de encendido del módulo alcanza el umbral de reinicio de 50 sin que se repita una avería.

Ayuda al diagnóstico
    • A veces, cuando está diagnosticando un problema de un cliente específico o después de una reparación, puede que vea que hay presente un código U de historial. Sin embargo, no hay un estado "actual" o "activo" asociado. Los códigos U de pérdida de comunicación como éstos se pueden establecer por diversas razones. Muchas veces resultan obvios para el operador y el técnico del vehículo o no tienen síntomas asociados. Eventualmente se puede borrar por sí mismos automáticamente después de una serie de ciclos de encendido sin fallos. Este problema se podría atribuir sobre todo a una de las situaciones siguientes:
       - Un módulo de control del circuito de comunicación de datos se desconectó cuando el circuito de comunicación estaba activo.
       - Durante el diagnóstico se interrumpió la alimentación de uno o más módulos.
       - Se produjo un problema de batería baja, de modo que varios módulos de control dejan de comunicarse cuando el voltaje de la batería cae por debajo de un cierto umbral.
       - Se restauró la alimentación de batería al vehículo y no todos los módulos de control del circuito de comunicación se reiniciaron al mismo tiempo.
       - Si aparece un código U de pérdida de comunicación en el historial sin una razón aparente, lo más probable es que esté asociado a una de las situaciones anteriores. Todos son problemas temporales y no se deberían interpretar nunca como una avería intermitente, haciendo que tenga sustituir una pieza.
    • Un módulo de control puede tener un código U almacenado en el historial que no requiera reparación alguna. Puede haber temporalmente problemas con mensajes tardíos o corruptos entre módulos de control sin un síntoma o avería aparente; no significa que el módulo de control esté averiado. No sustituya un módulo de control basándose sólo en un código U de historial.
    • No sustituya un módulo de control que informe sobre un código U. Éste identifica qué módulo de control se debe diagnosticar por un problema de comunicación.
    • Debe haber comunicación entre algún módulo y la herramienta de diagnóstico con el sistema de datos serie de GMLAN o CAN no operativo, ya sea de alta o baja velocidad. Esta condición se debe a que algún módulo utiliza ambos sistemas: GMLAN/CAN y KWP2000.
    • Use Referencias de enlace de datos para determinar si el módulo usa comunicaciones de datos de serie GMLAN de alta o baja velocidad.
    • Algunos módulos de control puede que no tengan protección interna para circuitos de control específicos y pueden abrir un fusible B+ o de encendido. Si está abierto un fusible y el circuito B+ o de encendido no está cortocircuitado a tierra, asegúrese de que ninguno de los circuitos de control están cortocircuitados a tierra antes de sustituir el módulo de control.
    • Este diagnóstico se puede usar para cualquier módulo de control que no se esté comunicando, independientemente del tipo de circuito de datos de serie al que esté conectado, siempre que el vehículo esté equipado con el módulo de control.

Comprobación del circuito/sistema

Nota: Use los esquemas para identificar lo siguiente:

    • Los módulos de control con los que está equipado el vehículo
    • Las ubicaciones del módulo de control en los circuitos de datos de serie GMLAN de alta y baja velocidad
    • Los módulos de control B+, encendido, tierra, habilitación de comunicación y terminales de circuitos de datos de serie
  1. Determine el módulo que no se está comunicando. Consultar Lista de códigos U del módulo de control .
  2. Con el encendido OFF, desconecte el conector del mazo de cables necesario del módulo que no se está comunicando.
  3. Pruebe con menos de 10 Ω entre cada terminal del circuito de masa y masa.
    4. ⇒Si es superior al intervalo especificado, compruebe si el circuito de masa tiene una resistencia abierta/alta.
  4. Si está equipada, verifique que la lámpara de prueba se ilumina entre cada terminal del circuito B+ y la toma de tierra.
    5. ⇒Si la luz de prueba no se ilumina, pruebe el circuito B+ para ver si hay un cortocircuito a tierra o una resistencia abierta/alta. Si el fusible del circuito está abierto, compruebe si los circuitos de control del módulo de control están cortocircuitados a tierra. Si la prueba de circuitos sale bien, sustituya el módulo de control.
  5. Si está equipado, encendido ON, verifique que la luz de prueba entre cada terminal del circuito de encendido y toma de tierra.
    6. ⇒Si la luz de prueba no se ilumina, compruebe si en el circuito de encendido hay un cortocircuito a masa o una resistencia abierta/alta. Si el fusible del circuito está abierto, compruebe si los circuitos de control del módulo de control están cortocircuitados a tierra. Si la prueba de circuitos sale bien, sustituya el módulo de control.

    Nota: Sólo los módulos GMLAN de alta velocidad tienen un circuito de habilitación de comunicación de datos de serie O un circuito de datos de serie de activación accesorio, y el BCM es la salida de estos circuitos, si procede. Consulte los esquemas de módulos para identificar qué módulos tienen estos circuitos.

  6. Si está equipada, con el encendido ON, verifique que la luz de prueba se ilumina entre el terminal del circuito de habilitación de comunicación O el terminal del circuito de activación accesorio y la toma de tierra.
    7. ⇒Si la luz de prueba no se ilumina, compruebe si el circuito tiene una resistencia abierta/alta. Si la prueba de circuitos sale bien, sustituya el BCM.
  7. Con el encendido OFF durante 60 segundos, pruebe con menos de 5 Ω entre los terminales del circuito de datos de serie y el terminal DLC indicado a continuación:
    8. • Terminal 2 del circuito de datos de serie de clase 2
    • Terminal 7 ó 8 del circuito de datos serie Keyword 2000
    • Terminal 1 de datos de serie GMLAN de baja velocidad
    • Terminal 6 ó 14 de datos de serie GMLAN de alta velocidad
    • Terminal 12 ó 13 del circuito de datos serie GMLAN de alta velocidad del bus de prolongación del chasis
    ⇒Si es superior al intervalo especificado, compruebe si el circuito de datos de serie tiene una resistencia abierta/alta entre el módulo de control que no se comunica y un módulo que establezca el DTC o un paquete de empalmes de datos de serie.
  8. Si todos los circuitos están bien, sustituya el módulo de control que no se está comunicando.
