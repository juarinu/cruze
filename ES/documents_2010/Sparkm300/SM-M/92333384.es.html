Descripción y funcionamiento de las comunicaciones de enlace de datos

Descripción de circuito

La comunicación entre los módulos de control se realiza a través del circuito de datos serie GMLAN de alta velocidad, el circuito de datos serie CAN y el circuito de datos serie Keyword 2000. Los módulos que necesitan comunicación en tiempo real se acoplan a la red GMLAN o CAN de alta velocidad. Algunos módulos de control utilizan la red GMLAN/CAN para comunicarse entre ellas mientras utilizan un circuito de datos serie Keyword 2000 independiente para la comunicación con la herramienta de diagnóstico. Consulte para obtener una lista de módulos y los buses con los que se comunican.

Descripción del circuito GMLAN de alta velocidad

El conector de enlace de datos (DLC) permite que una herramienta de diagnóstico comunique con el circuito de datos de serie GMLAN de alta velocidad. Los datos serie se transmiten en dos cables trenzados que permiten velocidades de hasta 500 Kb/s. El par trenzado termina en resistencias de 120 Ω. Las resistencias se utilizan para reducir el ruido en el bus GMLAN de alta velocidad durante el funcionamiento normal del vehículo. El bus GMLAN de alta velocidad es un bus diferencial. El bus (+) de datos de serie GMLAN de alta velocidad y el bus (-) de datos de serie GMLAN de alta velocidad se excitan en extremos opuestos desde un nivel de reposo o ralentí. El nivel de ralentí, que es de aproximadamente 2,5 V, se considera como datos transmitidos recesivos y se interpreta como una lógica 1. Excitar las líneas en sus extremos, añade un voltio en el bus (+) de datos de serie GMLAN de alta velocidad y resta un voltio del cable de bus (-) de datos de serie GMLAN de alta velocidad. Este estado dominante se interpreta como una lógica 0. La gestión de red GMLAN admite una configuración selectiva y se basa en redes virtuales. Una red virtual es una colección de señales iniciadas como respuesta a un evento en el vehículo. El inicio de una red virtual significa que se ha solicitado un aspecto concreto de la funcionalidad del vehículo. Una red virtual está sustentada por dispositivos virtuales que representan una colección de señales reguladas por un único dispositivo físico. Así, un dispositivo físico puede tener uno o varios dispositivos virtuales. La supervisión de señales es el proceso de determinar si se está recibiendo o no una señal esperada. La compensación de fallos es la capacidad de sustituir una señal por un valor por defecto o un algoritmo por defecto en ausencia de una señal válida. Algunos mensajes también se interpretan como un signo de actividad del dispositivo virtual. Si se pierde una señal de este tipo, la aplicación activará un código de no comunicación en relación con el dispositivo virtual correspondiente.

Nota: Una pérdida del DTC de datos serie no significa un fallo del módulo en el que se registra.

Descripción del circuito del Controlador de red de área (CAN)

La línea de datos serie del Controlador de red de área (CAN) no se comunica con la herramienta de diagnóstico a través del conector de enlace de datos (DLC) La línea de datos serie CAN es un bus de datos serie de alta velocidad utilizado para la comunicación de la información entre el módulo de control del motor (ECM) y el módulo de control de la transmisión (TCM). Las velocidades normales de transmisión de datos deben ser lo suficientemente altas para garantizar que se mantenga la respuesta en tiempo real requerida.

El esquema de direccionamiento empleado con CAN asigna una etiqueta a todos los mensajes, cada uno de los cuales recibe un identificador exclusivo. El identificador clasifica el contenido del mensaje. Cada módulo procesa solamente aquellos mensajes cuyos identificadores se guardan en la lista de aceptación del módulo. Esto es una forma de filtrado de mensajes del CAN.

El identificador etiqueta tanto el contenido de datos como la prioridad del mensaje que se está enviando. Cada módulo puede empezar a transmitir sus datos más importantes en cuanto el bus quede libre. Cuando más de un módulo empieza a transmitir al mismo tiempo, se concede acceso en primer lugar al mensaje con mayor prioridad. Un módulo responde al fallo para conseguir acceso conmutando automáticamente al modo de recepción, entonces el módulo repite el intento de transmisión tan pronto como el bus queda libre de nuevo.

Descripción del circuito de datos serie Keyword 2000

Los protocolos del Keyword utilizan una única línea de cable de datos bidireccional entre el módulo y la herramienta de diagnóstico. La estructura de mensajes es de tipo solicitud y respuesta. El Keyword 2000 se utiliza sólo para diagnóstico de herramienta de diagnóstico. Los módulos no intercambian datos en estos sistemas.

Conector de enlace de datos (DLC)

El conector de enlace de datos (DLC) es un conector de 16 cavidades estandarizado. El diseño y la ubicación del conector vienen dados por un estándar de cable industrial y es necesario que prevea lo siguiente:

    • Pasador 1 - terminal de comunicación GMLAN de baja velocidad
    • Pasador 3 - terminal de bus (+) GMLAN de velocidad media
    • Pasador 4 - terminal de masa de potencia de herramienta de diagnóstico
    • Pasador 5 - terminal de masa de señal común
    • Pasador 6 - terminal de bus (+) de datos de serie GMLAN de alta velocidad
    • Terminal del bus de datos serie del Keyword 2000 pasador 7
    • Terminal del bus de datos serie del Keyword 2000 pasador 8
    • Pasador 11 - terminal de bus (-) de datos de serie GMLAN de velocidad media
    • Pasador 12 - terminal de bus de datos de serie GMLAN de alta velocidad del chasis
    • Pasador 13 - terminal de bus de datos de serie GMLAN de alta velocidad del chasis
    • Pasador 14 - terminal de bus (-) de datos de serie GMLAN de alta velocidad
    • Pasador 16 - terminal de tensión positiva de la batería, corriente de herramienta de diagnóstico

Referencia de datos de serie

La herramienta de diagnóstico comunica a través de los distintos buses del vehículo. Cuando se instala una herramienta de diagnóstico en un vehículo, la herramienta de diagnóstico intentará comunicar con cualquier módulo con el que tenga opción en el vehículo. Si una opción no está instalada en el vehículo, la herramienta de diagnóstico mostrará el mensaje "No comunicación" para ese módulo de control de opciones. Para impedir fallos de diagnóstico de no comunicación con un módulo específico, consulte en Referencias de enlace de datos una lista de módulos, los buses con los que se comunican y los códigos RPO para un módulo específico.
