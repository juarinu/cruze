Descripción de DTC

DTC B101D XX: Hardware ECU

Descripción del circuito/sistema

La detección de fallos internos se realiza dentro del módulo de control. El número de síntoma indicado en la Descripción de DTC sólo es una referencia para los ingenieros. No está relacionado con ningún diagnóstico de circuito externo.

Condiciones para el funcionamiento de DTC

El módulo ejecuta el programa para detectar fallos internos cuando se ejecuta el encendido. Los únicos requisitos son tensión y masa. El programa se ejecuta incluso si la tensión se encuentra fuera del intervalo operativo.

Condiciones para ajustar el DTC

El módulo ha detectado un fallo de funcionamiento interno.

Acción realizada cuando se registra el DTC

El módulo no permite realizar entradas.

Condiciones para el borrar el DTC
    • Un DTC actual se borra cuando ya no existe la avería.
    • Un DTC del historial se borra cuando el contador del ciclo de encendido del módulo alcanza el umbral de reinicio de 50 sin que se repita una avería.

Ayuda al diagnóstico
    • Este DTC puede almacenarse como un DTC de historial sin afectar al funcionamiento del módulo de control.
    •  No sustituya un módulo de control basándose solamente en que se active un DTC B101D en el historial con la excepción de los siguientes módulos de control:
       - Módulo de diagnóstico y percepción del airbag (SDM) K36
       - Módulo de detección de presencia del pasajero K85
       - Módulo de control del calefactor de refrigerante K10
    • Si el DTC B101D se activa como actual, sustituya el módulo de control apropiado.

Verificación del circuito/sistema

Compruebe que no se ha activado el DTC B101D.

⇒Si se ha activado el DTC, sustituya el módulo que ha activado el DTC.
