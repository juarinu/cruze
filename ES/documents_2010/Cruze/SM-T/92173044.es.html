Descripción y funcionamiento de las comunicaciones de enlace de datos

Descripción de circuito

La comunicación entre los módulos de control se realiza a través del circuito de datos de serie GMLAN de alta velocidad, el circuito de datos de serie GMLAN de alta velocidad del chasis, el circuito de datos de serie GMLAN de velocidad media y el circuito de datos de serie GMLAN de baja velocidad. Los módulos que necesitan comunicación en tiempo real se acoplan a la red GMLAN de alta velocidad y la red GMLAN de alta velocidad del chasis. El módulo de control de la carrocería (BCM) es la pasarela entre las redes. El objetivo de la pasarela es traducir mensajes de datos de serie entre el bus GMLAN de alta velocidad y el bus GMLAN de baja velocidad. La pasarela interactuará con cada una de las redes según el protocolo de transmisión de la red. Consulte en Descripción y funcionamiento del sistema de control de la carrocería más información acerca de la pasarela.

Descripción del circuito GMLAN de alta velocidad

El conector de enlace de datos (DLC) permite que una herramienta de diagnóstico comunique con el circuito de datos de serie GMLAN de alta velocidad. Los datos de serie se transmiten en dos cables trenzados que permiten velocidades de hasta 500 Kb/s. El par trenzado termina en resistencias de 120 ohmios. Las resistencias se utilizan para reducir el ruido en el bus GMLAN de alta velocidad durante el funcionamiento normal del vehículo. El bus GMLAN de alta velocidad es un bus diferencial. El bus (+) de datos de serie GMLAN de alta velocidad y el bus (-) de datos de serie GMLAN de alta velocidad se excitan en extremos opuestos desde un nivel de reposo o ralentí. El nivel de ralentí, que es de aproximadamente 2,5 voltios, se considera como datos transmitidos recesivos y se interpreta como una lógica 1. Excitar las líneas en sus extremos, añade un voltio en el bus (+) de datos de serie GMLAN de alta velocidad y resta un voltio del cable de bus(-) de datos de serie GMLAN de alta velocidad. Este estado dominante se interpreta como una lógica 0. La gestión de red GMLAN admite una configuración selectiva y se basa en redes virtuales. Una red virtual es una colección de señales iniciadas como respuesta a un evento en el vehículo. El inicio de una red virtual significa que se ha solicitado un aspecto concreto de la funcionalidad del vehículo. Una red virtual está sustentada por dispositivos virtuales que representan una colección de señales reguladas por un único dispositivo físico. Así, un dispositivo físico puede tener uno o varios dispositivos virtuales. La supervisión de señales es el proceso de determinar si se está recibiendo o no una señal esperada. La compensación de fallos es la capacidad de sustituir una señal por un valor por defecto o un algoritmo por defecto en ausencia de una señal válida. Algunos mensajes también se interpretan como un signo de actividad del dispositivo virtual. Si se pierde una señal de este tipo, la aplicación activará un código de no comunicación en relación con el dispositivo virtual correspondiente. Tenga en cuenta que un DTC de pérdida de datos serie no representa un fallo del módulo en el cual está puesto el código.

Descripción del circuito GMLAN de alta velocidad del chasis

El bus GMLAN de alta velocidad de expansión del chasis funciona igual que el bus GMLAN de alta velocidad, y los dos buses funcionan en paralelo. El bus de expansión del chasis se añadió para reducir la congestión de mensajes en el bus de alta velocidad primario. Dado que el bus GMLAN de alta velocidad del chasis y el bus GMLAN de alta velocidad primario operan de la misma manera, los diagnósticos para ambos son iguales.

Descripción de circuito GMLAN de velocidad media

El conector de enlace de datos (DLC) permite que una herramienta de diagnóstico comunique con el circuito de datos de serie GMLAN de velocidad media. Los módulos conectados al circuito de datos serie GMLAN de velocidad media controlan comunicaciones de datos serie durante el funcionamiento normal del vehículo. Entre los módulos se intercambia información y comandos operativos cuando el interruptor de encendido está en cualquier posición distinta de OFF. El bus de datos de serie GMLAN de velocidad media utiliza una resistencia de terminación que está en paralelo a los circuitos GMLAN (+) y (-) de velocidad media.

Descripción de circuito GMLAN de baja velocidad

El conector de enlace de datos (DLC) permite que una herramienta de diagnóstico comunique con el circuito de datos de serie GMLAN de baja velocidad. Los datos de serie se transmiten por un cable simple a los módulos de control adecuados. La velocidad de transmisión para GMLAN de baja velocidad es de hasta 83,33 Kb/s. En circunstancias operativas normales del vehículo, la velocidad del bus es de 33,33 Kb/s. Este protocolo produce un único tren de impulsos enviado a través del bus de datos de serie GMLAN de baja velocidad. Si un módulo eleva el bus, 5 voltios, esto crea un estado de lógica dominante ó 0 en el bus. Si el bus se baja, 0 voltios, se traduce como un estado de lógica recesiva o 1. Para reactivar los módulos de control conectados al bus de datos de serie GMLAN de baja velocidad, se envía un impulso de reactivación de alta tensión a través del bus, el nivel de tensión del impulso es de +10 voltios. Los módulos conectados al bus GMLAN de baja velocidad pueden formar parte de una red virtual, tal como se ha referido en el párrafo anterior. Los módulos del bus de datos de serie GMLAN de baja velocidad están conectados al bus en una configuración en paralelo.

Descripción de circuito LIN (Local Interconnect Network)

Los datos de serie se transmiten por un bus de circuito de red de cable simple LIN (Local Interconnect Network) entre un módulo de control primario y otros módulos dentro de un subsistema particular. Si se pierde la comunicación de datos de serie entre cualquiera de los módulos de control en la red de buses LIN, se activará un código de no comunicación en relación con el módulo de control que no comunica.

Descripción de circuito de la Interfaz gráfica CAN (CGI)

La interfaz gráfica CAN (CGI) es una interfaz de datos de serie de alta velocidad basada en UART que utiliza una capa física CAN. El subsistema de entretenimiento la utiliza para transferir gráficos de pantalla de alta velocidad entre la radio y el módulo de la pantalla de información y/o el control HVAC/Radio. Cuando el módulo de la pantalla de información y el control HVAC/Radio son módulos de control independientes, la radio sólo interactúa con el módulo de la pantalla de información y éste se comunica con el control HVAC/Radio a través de una interfaz LIN. Por consiguiente, el módulo de la pantalla de información es responsable de pasar información entre la radio y el control HVAC/Radio. La radio o el módulo de la pantalla de información generarán una señal de activación del bus cuando se requiera la funcionalidad de los sistemas. Una vez activada la red CGI, la radio inicia todas las transferencias de mensajes del CGI. Los dispositivos subordinados como el módulo de la pantalla de información y el control HVAC/Radio puede que no transmitan a menos que estén respondiendo a una solicitud de la radio.

Las redes CGI de doble cable tienen una resistencia de terminación entre los conductores CGI+ (CAN- Alta) y CGI- (CAN- Baja) en cada extremo del bus. Cada resistencia de terminación es de 120 ohms.

La función de comunicación del CGI deberá estar habilitada o inhabilitada en función del nivel de tensión de una entrada dedicada (normalmente llamada "Activación de la pila central"). La red permanecerá activada siempre que la señal discreta esté ajustada baja a menos de 1,5 V. Cuando la radio determine que el bus de comunicación necesita estar dispuesto para el servicio o activo, generará la señal de activación tirando del circuito de señal bajo.

La radio puede ejecutar un reajuste en caliente del módulo de la pantalla de información si éste no responde a la solicitud de la radio mediante la interfaz CGI. El reajuste de la pila central es una salida de disminución de verificación baja (inferior a 1,5 V) de la radio al módulo de la pantalla de información y tiene las mismas características eléctricas que los de la señal de Activación de la pila central que se han definido anteriormente.

Conector de enlace de datos (DLC)

El conector de enlace de datos (DLC) es un conector de 16 cavidades estandarizado. El diseño y la ubicación del conector vienen dados por un estándar de cable industrial y es necesario que prevea lo siguiente:

    • Pasador 1 - terminal de comunicación GMLAN de baja velocidad
    • Pasador 3 - terminal de bus (+) GMLAN de velocidad media
    • Pasador 4 - terminal de masa de potencia de herramienta de diagnóstico
    • Pasador 5 - terminal de masa de señal común
    • Pasador 6 - terminal de bus (+) de datos de serie GMLAN de alta velocidad
    • Pasador 11 - terminal de bus (-) de datos de serie GMLAN de velocidad media
    • Pasador 12 - terminal de bus de datos de serie GMLAN de alta velocidad del chasis
    • Pasador 13 - terminal de bus de datos de serie GMLAN de alta velocidad del chasis
    • Pasador 14 - terminal de bus (-) de datos de serie GMLAN de alta velocidad
    • Pasador 16 - terminal de tensión positiva de la batería, corriente de herramienta de diagnóstico

Referencia de datos de serie

La herramienta de diagnóstico comunica a través de los distintos buses del vehículo. Cuando se instala una herramienta de diagnóstico en un vehículo, la herramienta de diagnóstico intentará comunicar con cualquier módulo con el que tenga opción en el vehículo. Si una opción no está instalada en el vehículo, la herramienta de diagnóstico mostrará el mensaje "No comunicación" para ese módulo de control de opciones. Para impedir fallos de diagnóstico de no comunicación con un módulo específico, consulte en Referencias de enlace de datos una lista de módulos, los buses con los que se comunican y los códigos RPO para un módulo específico.
