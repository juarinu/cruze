Descripción de DTC

Códigos de avería U1405: Datos no válidos recibidos del interruptor de control de velocidad

Descripción del circuito/sistema

El módulo de control de la carrocería (BCM) envía datos al módulo de control del motor (ECM). Los datos se envían al ECM a través de una red de comunicación denominada red de área local (Local Area Network, LAN). Se utilizan dos circuitos para transmitir datos LAN entre el BCM y el TCM. Una avería en la red LAN no hará por sí misma que se active el DTC U1405. Si se produce una avería de LAN, se activarán otros DTC además del DTC U1405.

Si el ECM recibe una transmisión de datos inválidos del BCM, este DTC se activa.

Condiciones para la activación de DTC

El interruptor de encendido está en Activ.

Condiciones para ajustar el DTC

El ECM no recibe datos válidos del BCM.

Acción realizada cuando el DTC se ajusta
    • La lámpara de indicación de realizar el mantenimiento pronto (SVS) se encenderá tan pronto como se produzca el error.
    • El ECM registrará las condiciones de servicio en el momento en que el diagnóstico falla. Esta información se almacenará en las memorias intermedias de registros de averías.
    • Se guarda un DTC de historial.

Condiciones para el borrar el DTC
    • La lámpara SVS se apagará después de que el ciclo de conducción actual en que se produjo el diagnóstico se ejecute sin ninguna avería.
    • Se borrará un historial de DTC una vez que se hayan producido 20 ciclos de calentamiento consecutivos sin fallos.
    • Los DTC se pueden borrar usando la herramienta de diagnóstico.
    • Desconexión de la alimentación de la batería del ECM durante 10 segundos.

Verificación del circuito/sistema

El DTC U1405 sólo se activa con otro DTC. El DTC U1405 se borrará tras diagnosticar el otro DTC. Consulte Lista de los códigos diagnósticos de avería (DTC) - Vehículo .
