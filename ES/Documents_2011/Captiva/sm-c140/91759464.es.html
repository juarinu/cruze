DTC U2100

Descripción de circuito

Los circuitos de datos en serie GMLAN son buses de datos en serie de red de área de controlador (CAN) que se usan para comunicar información entre los módulos de control. Las velocidades normales de transmisión de datos deben ser lo suficientemente altas para garantizar que se mantenga la respuesta en tiempo real requerida. En este vehículo hay 2 tipos totalmente distintos de circuitos de datos en serie GMLAN, el circuito de 2 hilos de alta velocidad y el circuito de hilo único de baja velocidad. Consulte Descripción y funcionamiento de las comunicaciones de enlace de datos para una descripción completa de estos buses. Los circuitos de datos en serie GMLAN también se comunican directamente con el conector DLC. Los mensajes son interpretados por el módulo CANdi externamente conectado, que actúa como transceptor para la herramienta de diagnóstico.

Los módulos conectados a los circuitos de datos en serie GMLAN monitorizan la comunicación durante el funcionamiento normal del vehículo, cuando información sobre el funcionamiento se intercambia entre módulos. Todos los módulos de las redes GMLAN mantienen un contador de errores de transmisión (TEC) y un contador de errores de recepción (REC). Los valores de contador aumentan con errores detectados y se reducen con mensajes sin errores. Si el valor TEC es superior a 255, el controlador se retira de la red y se activará un DTC U2100.

Descripción de DTC

Este procedimiento de diagnóstico soporta el siguiente DTC:

DTC U2100 00 Comunicación de bus CAN (Controller Area Network)

Condiciones para el funcionamiento de DTCs
    • La tensión de alimentación a los módulos de control está en el intervalo operativo normal.
    • El vehículo está en el modo de potencia MARCHA.

Condiciones para ajustar el DTC

El módulo que ha activado el DTC ha intentado establecer comunicaciones en los circuitos GMLAN en más de 7 ocasiones.

Acción realizada cuando los DTCs se ajustan
    • El módulo suspende la transmisión de todos los mensajes.
    • El módulo utiliza valores por defecto para todos los parámetros recibidos en los circuitos de datos de serie.
    • El módulo impide que se activen todos los demás códigos de avería de comunicación.

Condiciones para el borrar el DTC
    • Un DTC actual se borra cuando ya no existe la avería.
    • Un DTC del historial se borra cuando el contador del ciclo de encendido del módulo alcanza el umbral de reinicio de 50 sin que se repita una avería.

Ayuda al diagnóstico
    • Este DTC no puede recuperarse con un estado actual. El diagnóstico de un DTC actual se realiza mediante el síntoma, La herramienta de diagnóstico no se comunica con el dispositivo GMLAN de alta velocidad o La herramienta de diagnóstico no se comunica con el dispositivo GMLAN de baja velocidad, dependiendo de la comunicación de datos en serie del módulo.
    • Es probable que un estado intermitente se deba a un cortocircuito en los circuitos de datos de serie GMLAN. Use el procedimiento La herramienta de diagnóstico no se comunica con el dispositivo GMLAN de alta velocidad o La herramienta de diagnóstico no se comunica con el dispositivo GMLAN de baja velocidad para aislar una condición intermitente.
