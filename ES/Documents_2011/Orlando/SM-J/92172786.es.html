Descripción del circuito/sistema

Los datos de serie se transmiten por un cable simple a los módulos de control adecuados. La velocidad de transmisión para GMLAN de baja velocidad es de hasta 83,33 Kb/s. En circunstancias operativas normales del vehículo, la velocidad del bus es de 33,33 Kb/s. Los módulos de control conmutan el circuito de datos de serie entre 0-5 V en situaciones de comunicación normales. Para reactivar los módulos de control conectados al circuito de datos de serie GMLAN de baja velocidad se envía un impulso de reactivación de tensión de 10 V. Si se pierden datos de serie, los módulos de control activarán un código de no comunicación respecto al módulo de control que no comunica. Un DTC de pérdida de comunicación de datos de serie no significa un fallo del módulo que lo activa.

Ayuda al diagnóstico
    • A veces, cuando está diagnosticando un problema de un cliente específico o después de una reparación, puede que vea que hay presente un código U de historial. Sin embargo, no hay un estado "actual" o "activo" asociado. Los códigos U de pérdida de comunicación como éstos se pueden establecer por diversas razones. Muchas veces resultan obvios para el operador y el técnico del vehículo o no tienen síntomas asociados. Eventualmente se puede borrar por sí mismos automáticamente después de una serie de ciclos de encendido sin fallos. Este problema se podría atribuir sobre todo a una de las situaciones siguientes:
       - Un módulo de control del circuito de comunicación de datos se desconectó cuando el circuito de comunicación estaba activo.
       - Durante el diagnóstico se interrumpió la alimentación de uno o más módulos
       - Se produjo un problema de batería baja, de modo que varios módulos de control dejan de comunicarse cuando el voltaje de la batería cae por debajo de un cierto umbral.
       - Se restauró la alimentación de batería al vehículo y no todos los módulos de control del circuito de comunicación se reiniciaron al mismo tiempo.
       - Si aparece un código U de pérdida de comunicación en el historial sin una razón aparente, lo más probable es que esté asociado a una de las situaciones anteriores. Todos son problemas temporales y no se deberían interpretar nunca como una avería intermitente, haciendo que tenga sustituir una pieza.
    • No sustituya un módulo de control que informe sobre un código U. Éste identifica qué módulo de control se debe diagnosticar por un problema de comunicación.
    • Debe haber comunicación entre el módulo de control de la carrocería (BCM) y la herramienta de diagnóstico con el sistema de datos de serie de GMLAN de baja velocidad no operativo. Esto se debe a que el BCM usa tanto los sistemas GMLAN de alta como los de baja velocidad.
    • Una interrupción en el circuito de datos de serie GMLAN de baja velocidad entre el paquete de empalmes y un módulo sólo afectará a ese módulo específico. Este tipo de fallo activará un DTC de pérdida de comunicación para cada módulo afectado y los otros módulos seguirán comunicando.
    • Una interrupción en el terminal 5 de circuito de masa del conector de enlace de datos (DLC) permitirá que la herramienta de diagnóstico funcione, pero no comunicará con el vehículo.
    • Puede que el motor no arranque si existe un fallo total del circuito de datos en serie GMLAN de baja velocidad.
    • Los técnicos pueden encontrar varios códigos de avería (DTC) de comunicación LAN (Local Area Network) y ninguna comunicación LAN de baja velocidad con la herramienta de diagnóstico.
        Estas condiciones pueden deberse a la instalación a posteriori de un módulo de radio de navegación (véanse los boletines). Algunos clientes pueden comunicar uno o varios de los siguientes problemas:
       - El vehículo no se embala
       - El vehículo se embala, pero no arranca
       - Mensajes y luces de advertencia del sistema de mejora de la estabilidad del vehículo
       - Errores de posición de indicador de marcha PRNDL

Comprobación del circuito/sistema

Nota: Use los esquemas para identificar lo siguiente:

   • Los módulos de control con los que está equipado el vehículo
   • Ubicaciones del módulo de control y el paquete de empalmes en el circuito de datos en serie GMLAN de baja velocidad
   • Terminales de circuito de datos en serie GMLAN de baja velocidad para cada módulo de control, componente o paquete de empalmes

  1. Intente comunicarse con todos los módulos de control en el circuito de datos de serie GMLAN de velocidad baja. Consultar Referencias de enlace de datos . Debería no ser posible la comunicación con dos o más módulos de control en el circuito de datos de serie GMLAN de baja velocidad.
    2. ⇒Si únicamente un módulo de control no comunica, diagnostique únicamente ese módulo de control. Consultar DTC U0100-U02FF .
    ⇒Si uno o varios módulos de control comunican, pero no todos, consulte "comprobar si existe una resistencia alta/abierta en el circuito de datos de serie".
  2. Desconecte la herramienta de diagnóstico del DLC. Deberán realizarse las siguientes comprobaciones en el conector DLC.
  3. Con todas las puertas cerradas y el encendido en OFF durante 60 segundos, compruebe que existen menos de 10 Ω entre el terminal 5 de circuito a masa y la masa.
    4. ⇒Si es superior al intervalo especificado, compruebe si el circuito de masa tiene una resistencia abierta/alta.
  4. Con el encendido conectado, compruebe que existen menos de 4,5 V entre el terminal 1 de circuito de datos de serie y la masa.
    5. ⇒Si es superior al intervalo especificado, compruebe si el circuito de datos de serie presenta un cortocircuito con la tensión. Consulte "Comprobar si existe cortocircuito con la tensión en el circuito de datos de serie".
  5. Con el encendido en OFF durante 60 segundos, compruebe que existen más de 100 Ω entre el terminal del circuito de datos de serie 1 y masa.
    6. ⇒Si es inferior al intervalo especificado, compruebe si en el circuito de datos de serie existe un cortocircuito a masa. Consulte "comprobar si existe cortocircuito a masa en circuitos de datos de serie".
  6. Desconecte el conector del mazo de cables en el primer paquete de empalmes que está más cercano en el circuito al DLC.
  7. Compruebe que haya menos de 2 Ω entre el terminal 1 del circuito de datos en serie de DLC y el terminal de entrada de datos en serie del conector del paquete de empalmes.
    8. ⇒Si es mayor que el intervalo especificado, compruebe si el circuito de datos de serie tiene una resistencia abierta/alta. Si la comprobación del circuito es normal, sustituya el paquete de empalmes.

Comprobar si existe cortocircuito con la tensión en circuitos de datos de serie

  1. Con el encendido en OFF, desconecte los conectores correspondientes del mazo de cables en los siguientes componentes:
    2. • Todos los paquetes de empalmes de datos en serie a baja velocidad
    • Cualquier módulo o componente entre medio podría estar entre los paquetes de enlaces
  2. Con el encendido conectado, compruebe si existen menos de 4,5 V entre el terminal 1 de circuito de datos de serie del DLC y la masa.
    3. ⇒Si es superior al valor especificado, repare el cortocircuito con la tensión.
  3. Compruebe si existen menos de 4,5 V entre cada uno de los circuitos de datos en serie en el conector de cada uno de los componentes y masa.
    4. ⇒Si es superior al intervalo especificado, realice la comprobación adecuada indicada abajo:
    • En circuitos de datos de serie con un módulo de control, compruebe si el circuito de datos de serie presenta un cortocircuito con la tensión. Si las pruebas de circuito salen bien, sustituya el módulo de control.
    • En circuitos de datos de serie con 2 ó más módulos de control, compruebe cada una de las secciones del circuito de datos serie para ver si presenta un cortocircuito con la tensión. Si las comprobaciones de los circuitos son normales, sustituya el módulo de control que ocasiona el cortocircuito con la tensión al conectarse.

Comprobar si existe cortocircuito a masa en circuitos de datos de serie

  1. Con el encendido en OFF, desconecte los conectores correspondientes del mazo de cables en los siguientes componentes:
    2. • Todos los paquetes de empalmes de datos en serie a baja velocidad
    • Cualquier módulo o componente entre medio podría estar entre los paquetes de enlaces
  2. Compruebe si existe una resistencia infinita entre el terminal 1 de circuito de datos de serie del DLC y la masa.
    3. ⇒Si no se corresponde con el valor especificado, repare el cortocircuito a masa.
  3. Con el encendido en OFF durante 60 segundos, compruebe que existen más de 100 Ω entre cada uno de los circuitos de serie en el paquete de empalmes y la masa.
    4. ⇒Si es inferior al intervalo especificado, realice la comprobación adecuada indicada abajo:
    • En circuitos de datos de serie con un módulo de control, compruebe si el circuito de datos de serie presenta un cortocircuito a masa. Si las pruebas de circuito salen bien, sustituya el módulo de control.
    • En circuitos de datos de serie con 2 o más módulos de control, compruebe cada una de las secciones del circuito de datos serie para ver si presenta un cortocircuito a masa. Si las comprobaciones de los circuitos son normales, sustituya el módulo de control que ocasiona el cortocircuito a masa al conectarse.

Comprobar si existe una resistencia alta/abierta en el circuito de datos de serie.

  1. Si cuenta con un segundo paquete de empalmes, con el encendido conectado, compruebe que la herramienta de diagnóstico comunica con uno o varios módulos de control conectados al segundo paquete de empalmes.
    2. ⇒Si ninguno de los módulos de control en el segundo paquete de empalmes comunica, compruebe si existe una resistencia abierta/alta entre el primer y el segundo paquete de empalmes. Si las comprobaciones de los circuitos son normales, compruebe si el segundo paquete de empalmes presenta una resistencia abierta/alta. Si la comprobación del paquete de empalmes es normal, sustituya cualquier módulo o componente que pueda estar entre estos dos paquetes de empalmes.
  2. Si cuenta con un tercer paquete de empalmes, con el encendido conectado, compruebe que la herramienta de diagnóstico comunica con uno o varios módulos de control conectados al tercer paquete de empalmes.
    3. ⇒Si ninguno de los módulos de control en el tercer paquete de empalmes comunica, compruebe si existe una resistencia abierta/alta entre el segundo y el tercer paquete de empalmes. Si las comprobaciones de los circuitos son normales, compruebe si el tercer paquete de empalmes presenta una resistencia abierta/alta. Si la comprobación del paquete de empalmes es normal, sustituya cualquier módulo o componente que pueda estar entre estos dos paquetes de empalmes.
  3. Con el encendido desactivado, desconecte el paquete de empalmes que contiene los circuitos de datos en serie que van a los módulos que no comunican.
  4. Monte un cable puente con fusible de 3 A entre el terminal de entrada de datos en serie del conector de paquete de empalmes, y el circuito de datos en serie GMLAN de baja velocidad que no comunica:
  5. Con el encendido conectado, compruebe que la herramienta de diagnóstico comunica con los módulos de control conectados con el circuito de datos de serie GMLAN de baja velocidad.
    6. ⇒Si ningún módulo de control comunica, compruebe si existe una resistencia abierta/alta en cada una de las secciones del circuito de datos de serie. Si las comprobaciones de los circuitos son normales, sustituya el módulo de control que no comunica al conectarse.
