DTC U0073 o U2100

Instrucciones de diagnóstico
    • Lleve a cabo Comprobación del sistema de diagnóstico - Vehículo antes de usar el procedimiento de diagnóstico.
    • Revise Diagnosis basada en estrategia para tener una panorámica del enfoque de diagnóstico.
    • Instrucciones de procedimiento del diagnóstico proporciona una panorámica de cada categoría de diagnóstico.

Descripciones de DTC

Códigos de avería U0073 00: Bus de comunicación con el módulo de control desactivado

Códigos de avería U0073 71: Bus HS ECU desactivado

Códigos de avería U0073 72: Bus LS ECU desactivado

Códigos de avería U2100 00: Comunicación de bus CAN (Controller Area Network)

Códigos de avería U2100 47: Comunicación de bus CAN (Controller Area Network)

Descripción del circuito/sistema

Los circuitos de datos de serie son buses de datos de serie utilizados para comunicar información entre los módulos de control. Los circuitos de datos de serie también se conectan directamente al conector de enlace de datos (DLC).

Condiciones para el funcionamiento de DTCs
    • La tensión de alimentación en los módulos de control está en el intervalo operativo normal.
    • El modo de potencia del vehículo requiere comunicaciones de datos serie.

Condiciones para ajustar el DTC

El módulo de control que ha activado el DTC ha intentado establecer comunicaciones en los circuitos de datos en serie en más de 3 ocasiones en 5 segundos.

Acción realizada cuando los DTCs se ajustan
    • El módulo de control suspende la transmisión de todos los mensajes.
    • El módulo de control utiliza valores por defecto para todos los parámetros recibidos en los circuitos de datos en serie.
    • En el TCM, el DTC U0073 hará que la caja de cambios se ponga en marchas predeterminadas.
    • En el ECM y TCM, el DTC U0073 causará que se ilumine el testigo luminoso de avería (MIL).
    • El módulo de control impide que se activen todos los demás DTC de comunicación.

Condiciones para el borrar el DTC
    • Los módulos ECM/TCM apagan el MIL al cabo de 4 ciclos de encendido consecutivos en que el diagnóstico se realiza sin fallar.
    • Un DTC actual se borra cuando ya no existe la avería.
    • Un DTC del historial se borra cuando el contador del ciclo de encendido del módulo alcanza el umbral de reinicio de 50 sin que se repita una avería.

Verificación del circuito/sistema
  1. Consulte Referencias de enlace de datos para determinar el sistema de datos en serie que se utiliza para un módulo de control determinado.
  2. Este DTC no puede recuperarse con un estado actual. El diagnóstico se realiza utilizando el síntoma. Consultar Síntomas - Comunicaciones de datos .
