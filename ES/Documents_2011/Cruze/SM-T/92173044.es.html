Descripción y funcionamiento de las comunicaciones de enlace de datos

Nota: Ésta es una vista de conjunto de los diferentes buses de datos de serie usados por los módulos de control de GM para comunicar entre sí. Utilice Esquemas de comunicación de datos para averiguar los buses de datos de serie que están configurados para un vehículo específico.

Descripción de circuito

Hay muchos componentes en un vehículo que confían en la información de otros orígenes, o transmiten información a otros orígenes, o ambos. Las redes de comunicación de datos de serie proporcionan un medio fiable y con buena relación coste/eficacia para que diversos componentes del vehículo se comuniquen entre sí para compartir información.

GM utiliza una serie de diferentes buses de comunicación para garantizar el intercambio eficaz y justo a tiempo de información entre módulos de control. Cuando se comparan entre sí, algunos de estos buses son de diferente naturaleza en cuanto a velocidad, características de señal y comportamiento. Sirvan como ejemplo los buses GMLAN de alta velocidad y GMLAN de baja velocidad.

Por otra parte, cuando se comparan otros buses entre sí, tienen diferentes características y funcionan simplemente en paralelo. En este caso se usan para agrupar componentes que tienen una gran interacción. Sirvan como ejemplo los buses GMLAN de alta velocidad, de expansión del tren de rodaje y de expansión del chasis. Esto les permite comunicar entre sí en un bus con congestión de mensajes reducida, garantizando la información más rápida y justo a tiempo que si todos los módulos de control se encontrasen en un solo bus.

La mayoría de la información existente en el interior de una red dada suele mantenerse a nivel local, sin embargo puede ser necesario compartir alguna información con otras redes. Los módulos de control designados como pasarela desempeñan la función de transferir información entre los distintos buses. Un módulo de pasarela está conectado a un mínimo de 2 buses, e interaccionará con cada red según su estrategia de mensajes y sus modelos de transmisión.

El GMLAN proporciona la capacidad para que un módulo de control de recepción supervise las transmisiones de mensajes procedentes de otros módulos de control, al objeto de determinar si no se están recibiendo mensajes de interés. El objetivo primario consiste en permitir la sustitución de valores predeterminados razonables por la información que ya no se está recibiendo. Además, un módulo de control puede generar un código diagnóstico de avería para indicar que ya no funciona el módulo de control del cual está esperando recibir información.

Descripción del circuito GMLAN de alta velocidad

Un bus GMLAN de alta velocidad se usa cuando es necesario intercambiar datos a una velocidad suficientemente alta para minimizar el retardo entre la ocurrencia de un cambio de valor de sensor y la recepción de esa información por un dispositivo de control que utiliza la información para ajustar las prestaciones del sistema del vehículo.

La red GMLAN de datos de serie de alta velocidad consiste en un par de cables trenzados. Un circuito de señal se identifica como GMLAN alto y el otro circuito de señal se identifica como GMLAN bajo. En cada extremo del bus de datos hay un resistor terminal de 120 Ω entre los circuitos GMLAN alto y GMLAN bajo.

Los símbolos de datos (1 y 0) se transmiten secuencialmente a una velocidad de 500 Kbit/s. Los datos que se han de transmitir por el bus se representan mediante la diferencia de tensión entre la tensión de señal del GMLAN alto y la tensión de señal del GMLAN bajo.

Cuando el bus de dos cables se encuentra en reposo, no se excitan los circuitos de señal de GMLAN alto y de GMLAN bajo, lo cual representa una lógica "1". En este estado, ambos circuitos de señal se encuentran a la misma tensión de 2,5 V. La tensión diferencial es de aproximadamente 0 V.

Cuando se va a transmitir una lógica "0", se excita el circuito de señal de GMLAN hacia arriba a más de 3,5 V aproximadamente, mientras que el circuito de GMLAN bajo se excita en descenso, a 1,5 V aproximadamente. La tensión diferencial es de aproximadamente 2,0 (+/- 0,5) V.

Descripción del circuito GMLAN de alta velocidad del chasis

El bus GMLAN de expansión del chasis es básicamente una copia del bus GMLAN de alta velocidad, salvo que su uso está reservado a los componentes del chasis. Esta implementación divide la congestión de mensajes entre dos buses paralelos, ayudando a garantizar la transmisión y recepción de mensajes justo a tiempo. A veces, se necesita comunicación entre el bus de expansión del chasis y el bus GMLAN de alta velocidad principal. Esto se consigue usando el módulo de control electrónico de frenos (EBCM) como módulo de pasarela. Dado que el bus de expansión del chasis GMLAN de alta velocidad y el bus GMLAN de alta velocidad principal operan de la misma manera, los diagnósticos para ambos son similares.

Descripción de circuito de la Interfaz gráfica CAN (CGI)

El subsistema de entretenimiento utiliza este bus para transferir gráficos de pantalla de alta velocidad entre la radio y el módulo de la pantalla de información y/o el control HVAC/Radio. Las características eléctricas del bus CAN de interfaz gráfica (CGI) son muy similares a las del bus GMLAN de alta velocidad. No obstante, la estrategia y la construcción de los mensajes son diferentes. A veces, se necesita comunicación entre el bus CAN de interfaz gráfica y el bus GMLAN de baja velocidad. Esto se consigue usando la radio SilverBox como módulo de pasarela. Como el bus CAN de interfaz gráfica y el bus GMLAN de alta velocidad tienen características eléctricas similares, los diagnósticos son similares para ambos.

En el caso en que el módulo de la pantalla de información y el control de radio/HVAC sean módulos de control separados, el módulo de la pantalla de información es responsable de pasar la información entre la radio y el módulo de control de radio /HVAC. La radio sólo tiene interfaz con el módulo de pantalla de información, y el módulo de pantalla de información se comunica entonces con el control de radio/HVAC a través de una interfaz de red de interconexión local (LIN).

La radio o el módulo de la pantalla de información generarán una señal de activación del bus cuando se requiera la funcionalidad del sistema. La función de comunicación de la interfaz gráfica CAN se activará o se desactivará basándose en el nivel de tensión de activación de la pila central. La red permanecerá activada mientras la tensión del circuito esté excitada en bajo, a menos de 1,5 V. las comunicaciones se desactivan con un tensión de circuito alta de aproximadamente 5,0 V.

La radio puede ejecutar un reajuste en caliente del módulo de la pantalla de información si éste no responde a la solicitud de la radio. El reajuste de la pila central es una salida de disminución de verificación baja (inferior a 1,5 V) de la radio al módulo de la pantalla de información y tiene las mismas características eléctricas que los de la señal de activación de la pila central que se han definido anteriormente.

Descripción de circuito GMLAN de velocidad media

El bus GMLAN de media velocidad es muy parecido al bus GMLAN de alta velocidad, salvo que emplea una velocidad de transmisión más baja, de 125 Kbit/s. Este bus está concebido para usarlo donde el tiempo de respuesta del sistema requiere transmitir una gran cantidad de datos en un espacio de tiempo relativamente corto, como por ejemplo para actualizar una pantalla gráfica. Como tal, se ha usado con frecuencia para aplicaciones de infoentretenimiento. A veces, se necesita comunicación entre el bus GMLAN de baja velocidad y el bus GMLAN de media velocidad. Esto se consigue usando la radio (SilverBox) como módulo de pasarela. Dado que el bus GMLAN de media velocidad y el bus GMLAN de alta velocidad principal operan de forma similar, los diagnósticos para ambos son similares.

Descripción de circuito GMLAN de baja velocidad

El bus GMLAN de baja velocidad se usa en aplicaciones que requieren una alta velocidad de datos, con lo cual se pueden emplear componentes menos complejos. Se usa típicamente para funciones controladas por el operador, cuyos requisitos de tiempo de respuesta son más bajos que los necesarios para el control dinámico del vehículo.

La red de datos de serie GMLAN de baja velocidad consiste en un bus de un solo cable referenciado a masa, con excitación de tensión en el lado alto. Durante la operación del vehículo en la carretera, se transmiten símbolos de datos (1 y 0) de forma secuencial, a la velocidad normal de 33,3 Kbit/s. Se puede usar un modo especial de datos de alta velocidad, 83,3 Kbit/s exclusivamente para la programación de componentes.

A diferencia de las redes de doble cable de alta velocidad, la red de baja velocidad de un solo cable no utiliza resistores terminales en ambos extremos de la red.

Los símbolos de datos que se transmiten por el bus están representados por diferentes señales de tensión en el autobús. Cuando el bus GMLAN de baja velocidad se encuentra en reposo y no recibe excitación, hay una tensión de señal baja de aproximadamente 0,2 V. Esto representa una lógica "1". Cuando se va a transmitir una lógica "0", se excita la tensión de señal hacia arriba, a aproximadamente 4,0 V o más alta.

Descripción de circuito LIN (Local Interconnect Network)

El bus de red de interconexión local (LIN) consiste en un solo cable con una velocidad de transmisión de 10,417 Kbit/s. Este bus se usa para intercambiar información entre un módulo de control maestro y otros dispositivos inteligente que proporcionan funcionalidad de soporte. Este tipo de configuración no requiere la capacidad ni la velocidad de un bus GMLAN de alta velocidad ni de un GMLAN de baja velocidad, y por ello es relativamente más simple.

Los símbolos de datos (1 y 0) que se van a transmitir están representados por distintos niveles de tensión en el bus de comunicación. Cuando el bus LIN está en reposo y no excitado, la señal se encuentra en un estado de tensión alta de aproximadamente Vbat. Esto representa una lógica de "1". Cuando se va a transmitir una lógica "0", se excita la tensión de señal en descenso, a aproximadamente masa (0,0 V).

Descripción de circuito habilitado de comunicación

Los módulos de control en las redes del tipo GMLAN de alta velocidad, salvo el bus GMLAN de media velocidad y el bus CAN de interfaz gráfica, habilitan o deshabilitan la comunicación basándose en el nivel de tensión de este circuito. Cuando la tensión del circuito es alta (alrededor de 12 V), se habilitan las comunicaciones. Cuando el circuito es bajo, se deshabilitan las comunicaciones.

El bus CAN de interfaz gráfica es similar, pero emplea distintos niveles de tensión. Vea la descripción de arriba para el bus CAN de interfaz gráfica.

Conector de enlace de datos (DLC)

El conector de enlace de datos (DLC) es un conector de 16 cavidades estandarizado. El diseño y la ubicación del conector vienen dados por un estándar de cable industrial y es necesario que prevea lo siguiente:

    • Pasador 1 - terminal de comunicación GMLAN de baja velocidad
    • Pasador 2 clase 2 - terminal de comunicación
    • Pasador 3 - terminal de bus (+) GMLAN de velocidad media
    • Pasador 4 - terminal de masa de potencia de herramienta de diagnóstico
    • Pasador 5 - terminal de masa de señal común
    • Pasador 6 - terminal de bus (+) de datos de serie GMLAN de alta velocidad
    • Pasador 7 - terminal de comunicación de teclado
    • Pasador 11 - terminal de bus (-) de datos de serie GMLAN de velocidad media
    • Pasador 12 - terminal de bus (+) de datos de serie GMLAN de alta velocidad del chasis
    • Pasador 13 - terminal de bus (-) de datos de serie GMLAN de alta velocidad del chasis
    • Pasador 14 - terminal de bus (-) de datos de serie GMLAN de alta velocidad
    • Pasador 16 - terminal de tensión positiva de la batería, corriente de herramienta de diagnóstico

Referencia de datos de serie

La herramienta de diagnóstico comunica a través de los distintos buses del vehículo. Cuando se instala una herramienta de diagnóstico en un vehículo, la herramienta de diagnóstico intentará comunicar con cualquier módulo control con el que tenga opción en el vehículo. Si una opción no está instalada en el vehículo, la herramienta de diagnóstico mostrará el mensaje "No comunicación" para ese módulo de control opcional. Para impedir fallos de diagnóstico de no comunicación con un módulo de control específico, consulte en Referencias de enlace de datos una lista de módulos de control, los buses con los que se comunican y los códigos RPO para un módulo de control específico.
