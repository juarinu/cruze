Descripción del circuito/sistema

El bus de expansión GMLAN de alta velocidad del chasis funciona igual que el bus GMLAN de alta velocidad, y los dos buses funcionan en paralelo. El bus de expansión del chasis se añadió para reducir la congestión de mensajes en el bus de alta velocidad primario. Dado que el bus GMLAN de alta velocidad del chasis y el bus GMLAN de alta velocidad primario operan de la misma manera, los diagnósticos para ambos son iguales.

Los datos de serie se transmiten en dos cables trenzados que permiten velocidades de hasta 500 Kb/s. El par trenzado está terminado con dos resistencias de 120 ohmios, una es interna al módulo de control del motor (ECM) y la otra puede ser una resistencia independiente en un conjunto de conector o en otro módulo de control. Las resistencias se utilizan como la carga para el bus GMLAN de alta velocidad durante el funcionamiento normal del vehículo. El bus GMLAN de alta velocidad es un bus diferencial. El bus (+) de datos de serie GMLAN de alta velocidad y el bus (-) de datos de serie GMLAN de alta velocidad se excitan en extremos opuestos desde un nivel de reposo o ralentí de aproximadamente 2,5 V. Excitar las líneas en sus extremos, añade un voltio al circuito de bus (+) de datos de serie GMLAN de alta velocidad y resta un voltio del circuito de bus (-) de datos de serie GMLAN de alta velocidad. Si se pierden datos de serie, los módulos de control activarán un código de no comunicación respecto al módulo de control que no comunica. Tenga en cuenta que un DTC de pérdida de datos de serie no representa un fallo del módulo que lo activa.

Ayuda al diagnóstico
    • A veces, cuando está diagnosticando un problema de un cliente específico o después de una reparación, puede que vea que hay presente un código U de historial. Sin embargo, no hay un estado "actual" o "activo" asociado. Se pueden registrar códigos U de pérdida de comunicación como éstos por diversas razones. Muchas veces resultan obvios para el operador y el técnico del vehículo o no tienen síntomas asociados. Eventualmente se puede borrar por sí mismos automáticamente después de una serie de ciclos de encendido sin fallos. Este problema se podría atribuir sobre todo a una de las situaciones siguientes:
       - Un módulo de control del circuito de comunicación de datos se desconectó cuando el circuito de comunicación estaba activo.
       - Durante el diagnóstico se interrumpió la alimentación de uno o más módulos
       - Se produjo un problema de batería baja, de modo que varios módulos de control dejan de comunicarse cuando el voltaje de la batería cae por debajo de un cierto umbral.
       - Se restauró la alimentación de batería al vehículo y no todos los módulos de control del circuito de comunicación se reiniciaron al mismo tiempo.
       - Si aparece un código U de pérdida de comunicación en el historial sin una razón aparente, lo más probable es que esté asociado a una de las situaciones anteriores. Todos son problemas temporales y no se deberían interpretar nunca como una avería intermitente, haciendo que tenga sustituir una pieza.
    • No sustituya un módulo de control que comunica un código U. El código U identifica qué módulo de control es necesario diagnosticar en relación con un problema de comunicación.
    • Debe haber comunicación entre el BCM y la herramienta de diagnóstico con el sistema de datos de serie de GMLAN de alta velocidad no operativo. Esto se debe a que el BCM usa tanto los sistemas GMLAN de alta como los de baja velocidad.
    • Una interrupción en el terminal 5 de circuito de masa del DLC permitirá que la herramienta de diagnóstico funcione, pero no comunicará con el vehículo.
    • El motor no arrancará si existe un fallo total del bus de datos de serie GMLAN de alta velocidad.
    • Los técnicos pueden encontrar varios códigos de avería de comunicación LAN (Local Area Network) y ninguna comunicación GMLAN de baja velocidad con la herramienta de diagnóstico.
    • El motor no arrancará si existe un fallo total del bus de datos de serie GMLAN de alta velocidad.
        Estas condiciones pueden deberse a la instalación a posteriori de un módulo de radio de navegación (véanse los boletines). Algunos clientes pueden comunicar uno o varios de los siguientes problemas:
       - El vehículo no se embala
       - El vehículo se embala, pero no arranca
       - Mensajes y luces de advertencia del sistema de mejora de la estabilidad del vehículo
       - Errores de posición de indicador de marcha PRNDL

Comprobación del circuito/sistema

Nota: Puede ser necesario desconectar cada uno de los módulos de control para aislar un circuito defectuoso. Use los esquemas para identificar lo siguiente:

   • Los módulos de control con los que está equipado el vehículo
   • Resistencias de terminación de circuito de datos de serie GMLAN de alta velocidad del chasis
   • Las ubicaciones del módulo de control en los circuitos de datos de serie GMLAN de alta velocidad del chasis
   • Terminales de circuito de datos de serie GMLAN de alta velocidad del chasis de cada módulo de control

  1. Intente comunicarse con todos los módulos de control en el circuito de datos de serie GMLAN de alta velocidad. Consultar Referencias de enlace de datos . No debería ser posible la comunicación con dos o más módulos de control en el circuito de datos de serie GMLAN de alta velocidad del chasis.
    2. ⇒Si únicamente un módulo de control no comunica, diagnostique únicamente ese módulo de control. Consultar DTC U0100-U02FF .
  2. Desconecte la herramienta de diagnóstico del DLC. Deberán realizarse las siguientes comprobaciones en el conector DLC.
  3. Con todas las puertas cerradas y el encendido en OFF durante 60 segundos, compruebe que existen menos de 10 Ω entre el terminal 5 de circuito a masa y la masa.
    4. ⇒Si es mayor que el margen especificado, compruebe si hay un circuito abierto/resistencia alta en el circuito de masa.
  4. Con el encendido conectado, compruebe que existen menos de 4,5 V entre los circuitos de datos de serie indicados abajo y la masa:
    5. • Terminal 12
    • Terminal 13
    ⇒Si es superior al intervalo especificado, compruebe si el circuito de datos de serie presenta un cortocircuito con la tensión. Consulte Comprobar si existe cortocircuito con la tensión en circuitos de datos de serie
  5. Con el encendido en OFF durante 60 segundos, compruebe que existen más de 100 Ω entre los circuitos de datos de serie indicados abajo y la masa:
    6. • Terminal 12
    • Terminal 13
    ⇒Si no se corresponde con el valor especificado, compruebe si el circuito de datos de serie tiene un cortocircuito a masa. Consulte "comprobar si existe cortocircuito a masa en circuitos de datos de serie".
  6. Compruebe si existen 50-70 Ω entre el terminal 12 de circuito de datos de serie y el terminal 13.
    7. ⇒Si es inferior a 35 Ω, compruebe si existe un cortocircuito entre los circuitos de datos de serie. Consulte Comprobar si existe cortocircuito entre los circuitos de datos de serie
    ⇒Si existen 35-50 Ω, puede haber una tercera resistencia de terminación entre los circuitos de datos de serie. Esto puede suceder si se ha instalado un módulo de control incorrecto. Algunos módulos de control están disponibles con y sin resistencias de terminación instaladas para reducir la necesidad de resistencias de terminación en el mazo de cables.
    ⇒Si es superior a 70 Ω pero no es infinita, compruebe si el circuito de datos de serie tiene una resistencia alta/abierta. Consulte Comprobar si existe una resistencia alta/abierta en los circuitos de datos de serie.
    ⇒Si es infinita, compruebe si los circuitos de datos de serie tienen una resistencia abierta/alta entre el DLC y la primera conexión al circuito de datos de serie.

Comprobar si existe cortocircuito con la tensión en circuitos de datos de serie

  1. Con el encendido en OFF, desconecte los conectores del mazo de cables con los circuitos de datos de serie GMLAN de alta velocidad del chasis en un módulo de control de fácil acceso.
  2. Con el encendido conectado, compruebe que existen más de 4,5 V entre cada uno de los circuitos de datos de serie en el conector del módulo de control que acaba de desconectarse y la masa. Verifique si uno o varios circuitos de datos de serie son superiores a 4,5 V.
    3. ⇒Si todos los circuitos de datos de serie son inferiores al intervalo especificado, sustituya el módulo de control que se desconectó.
  3. Con el encendido en OFF, desconecte los conectores del mazo de cables con los circuitos de datos de serie GMLAN de alta velocidad del chasis en otro módulo de control en la dirección del circuito con cortocircuito con la tensión.
  4. Con el encendido conectado, compruebe que existen más de 4,5 V entre los circuitos de datos de serie en el conector del módulo de control que acaba de desconectarse y la masa. Verifique si uno o varios circuitos de datos de serie son superiores a 4,5 V.
    5. ⇒Si todos los circuitos de datos de serie son inferiores al intervalo especificado, sustituya el módulo de control que acaba de desconectarse.
  5. Repita el paso 3 hasta que se aisle una de las siguientes condiciones:
    6. • Un cortocircuito con la tensión en el circuito de datos de serie entre 2 módulos de control o paquetes de empalmes, si cuenta con éstos.
    • Un cortocircuito con la tensión en el circuito de datos de serie entre un módulo de control y una resistencia con terminación.

Comprobar si existe cortocircuito a masa en circuitos de datos de serie

  1. Con el encendido en OFF, desconecte los conectores del mazo de cables con los circuitos de datos de serie GMLAN de alta velocidad del chasis en un módulo de control de fácil acceso.
  2. Con el encendido en OFF durante 60 segundos, compruebe si existen menos de 1k Ω entre cada uno de los circuitos de datos de serie en el conector del módulo de control que acaba de desconectarse y la masa. Verifique si uno o varios circuitos de datos de serie son inferiores a 1k Ω.
    3. ⇒Si todos los circuitos de datos de serie son superiores al valor especificado, sustituya el módulo de control que se desconectó.
  3. Desconecte los conectores del mazo de cables con los circuitos de datos de serie GMLAN de alta velocidad en otro módulo de control en la dirección del circuito con cortocircuito a masa.
  4. Con el encendido en OFF durante 60 segundos, compruebe si existen menos de 1k Ω entre los circuitos de datos de serie en el conector del módulo de control que acaba de desconectarse y masa. Verifique si uno o varios circuitos de datos de serie son inferiores a 1k Ω.
    5. ⇒Si es superior al intervalo especificado para todos los circuitos de datos de serie, sustituya el módulo de control que se acaba de desconectar.
  5. Repita el paso 3 hasta que se aisle una de las siguientes condiciones:
    6. • Un cortocircuito a masa en el circuito de datos de serie entre 2 módulos de control o paquetes de empalmes, si cuenta con éstos.
    • Un cortocircuito a masa en el circuito de datos de serie entre un módulo de control y una resistencia de terminación.

Comprobar si existe cortocircuito entre los circuitos de datos de serie

  1. Con el encendido en OFF, desconecte los conectores del mazo de cables con los circuitos de datos de serie GMLAN de alta velocidad del chasis en un módulo de control de fácil acceso que no esté comunicando.
  2. Con el encendido en OFF durante 60 segundos, compruebe si existen menos de 35 Ω entre cada par de circuitos de datos de serie en el conector del módulo de control que acaba de desconectarse. Verifique que un par de circuitos de datos de serie son inferiores a 35 Ω.
    3. ⇒Si todos los pares de circuitos de datos de serie son superiores al valor especificado, sustituya el módulo de control que se desconectó.
  3. Conecte los conectores del mazo de cables en el módulo de control que se desconectó.
  4. Desconecte los conectores del mazo de cables con los circuitos de datos de serie GMLAN de alta velocidad del chasis en otro módulo de control en la dirección de los circuitos cortocircuitados juntos.
  5. Con el encendido en OFF durante 60 segundos, compruebe si existen menos de 35 Ω entre cada par de circuitos de datos de serie en el conector del módulo de control que acaba de desconectarse. Verifique que un par de circuitos de datos de serie son inferiores a 35 Ω.
    6. ⇒Si todos los pares de circuitos de datos de serie están dentro del intervalo especificado, sustituya el módulo de control que acaba de desconectarse.
  6. Repita el paso 3 hasta que se aisle una de las siguientes condiciones:
    7. • Circuitos de datos de serie cortocircuitados juntos entre 2 módulos de control o paquetes de empalmes, si cuenta con éstos.
    • Circuitos de datos de serie cortocircuitados juntos entre un módulo de control y una resistencia de terminación
    • Una resistencia de terminación cortocircuitada

Comprobar si existe una resistencia alta/abierta en los circuitos de datos de serie

  1. Con el encendido en OFF, desconecte los conectores del mazo de cables con los circuitos de datos de serie GMLAN de alta velocidad del chasis en un módulo de control de fácil acceso que no esté comunicando.
  2. Con el encendido en OFF durante 60 segundos, compruebe si existen más de 100k Ω entre cada par de circuitos de datos de serie en el conector del módulo de control que acaba de desconectarse. Compruebe si un par de circuitos de datos de serie es superior a 100k Ω.
    3. ⇒Si todos los pares de circuitos de datos de serie son inferiores al intervalo especificado, sustituya el módulo de control que se desconectó.
  3. Conecte los conectores del mazo de cables en el módulo de control que se desconectó.
  4. Desconecte los conectores del mazo de cables con los circuitos de datos de serie GMLAN de alta velocidad del chasis en otro módulo de control en la dirección del circuito abierto.
  5. Con el encendido en OFF durante 60 segundos, compruebe si existen más de 100k Ω entre cada par de circuitos de datos de serie en el conector del módulo de control que acaba de desconectarse. Compruebe si un par de circuitos de datos de serie es superior a 100k Ω.
    6. ⇒Si todos los pares de circuitos de datos de serie son inferiores al intervalo especificado, sustituya el módulo de control que acaba de desconectarse.
  6. Repita el paso 3 hasta que se aisle una de las siguientes condiciones:
    7. • Una resistencia alta/abierta en el circuito de datos de serie entre 2 módulos de control o paquetes de empalmes, si cuenta con éstos.
    • Una resistencia abierta/alta en el circuito de datos de serie entre un módulo de control y una resistencia de terminación
    • Una resistencia de terminación con resistencia abierta/alta
