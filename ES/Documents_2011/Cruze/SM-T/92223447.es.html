Definiciones del tipo de código diagnóstico de avería (DTC) de la cadena cinemática

DTC relacionados con las emisiones

Acción realizada cuando se registra el DTC - Tipo A

    • El módulo de control hace que se encienda el testigo luminoso de avería (MIL) cuando se realiza el diagnóstico y éste falla.
    • El módulo de control registra las condiciones de funcionamiento en el momento en que falló el diagnóstico. El módulo de control almacena esta información en instantáneas de datos/registros de averías.

Acción realizada cuando se registra el DTC - Tipo B

    • El módulo de control hace que se encienda el MIL cada segundo ciclo de encendido consecutivo en que se realiza el diagnóstico y éste falla.
    • El módulo de control registra las condiciones de funcionamiento en el momento en que falló el diagnóstico. La primera vez que falla el diagnóstico, el módulo de control anota esta información en los registros de averías. Si el diagnóstico informa de un fallo de funcionamiento en el segundo ciclo de encendido consecutivo, el módulo de control registra las condiciones de funcionamiento en el momento del fallo. El módulo de control escribe las condiciones de funcionamiento de la instantánea de datos y actualiza los registros de averías.
    • Lo siguiente es válido para los DTC de fallo de encendido:
       - Si el módulo de control detecta un nivel bajo o una condición de fallo de encendido en el nivel de emisiones durante 2 accionamientos consecutivos, hará que se encienda el MIL.
       - Si el módulo de control detecta un alto nivel o un fallo de encendido por daños en el catalizador, el módulo de control hace que el MIL parpadee con una velocidad de una intermitencia por segundo.
       - Si el módulo de control detecta un fallo de encendido durante 2 accionamientos no consecutivos, las condiciones almacenadas se comparan con las condiciones actuales. El módulo de control enciende el MIL cuando se dan las siguientes condiciones:
        • La carga del motor está a menos del 20% de la comprobación anterior que falló.
        • El régimen del motor está a menos de 375 r.p.m. de la comprobación anterior que falló.
        • La temperatura del refrigerante del motor se encuentra en el mismo margen que la comprobación anterior que falló.
    • Lo siguiente es válido para los DTC de ajuste de combustible:
       - Si el módulo de control detecta una condición de ajuste de combustible durante 2 accionamientos consecutivos, el módulo de control enciende el MIL.
       - Si el módulo de control detecta una condición de ajuste de combustible durante 2 accionamientos no consecutivos, las condiciones almacenadas se comparan con las condiciones actuales. El módulo de control enciende el MIL cuando se dan las siguientes condiciones:
        • La carga del motor está a menos del 20% de la comprobación anterior que falló.
        • El régimen del motor está a menos de 375 r.p.m. de la comprobación anterior que falló.
        • La temperatura del refrigerante del motor se encuentra en el mismo margen que la comprobación anterior que falló.

Condiciones para apagar o borrar el MIL/DTC - Tipo A o tipo B

    • El módulo de control apaga el MIL al cabo de 4 ciclos de encendido consecutivos en que el diagnóstico se realiza sin fallar.
    • Se borra un DTC actual, el de fallo en la última comprobación, cuando el diagnóstico se realiza sin errores.
    • Un DTC del historial se borra al cabo de 40 ciclos de calentamiento consecutivos si no se comunica ningún fallo de este diagnóstico o de cualquier otro relacionado con las emisiones.
    • Borre el MIL y el DTC con una herramienta de diagnóstico.

DTC no relacionados con las emisiones

Acción realizada cuando se registra el DTC - Tipo C

    • El módulo de control almacena la información de los DTC en la memoria cuando se realiza un diagnóstico y éste falla.
    • El MIL no se encenderá.
    • El módulo de control registra las condiciones de funcionamiento en el momento en que falló el diagnóstico. El módulo de control almacena esta información en los registros de averías.
    • El centro de información del conductor, si se dispone de uno, puede mostrar un mensaje.

Condiciones para borrar el DTC - Tipo C

    • Se borra un DTC actual, el de fallo en la última comprobación, cuando el diagnóstico se realiza sin errores.
    • Un DTC del historial se borra al cabo de 40 ciclos de calentamiento consecutivos si no se comunica ningún fallo de este diagnóstico o de cualquier otro no relacionado con las emisiones.
    • Borre los DTC con una herramienta de diagnóstico.

Acción realizada cuando se registra el DTC - Tipo D

    • El módulo de control realizará las acciones predeterminadas pero no se almacenará ningún DTC en los registros de averías.
    • El MIL o SVS no se encenderá.

Condiciones para borrar el DTC - Tipo D

Borre los DTC con una herramienta de diagnóstico.
