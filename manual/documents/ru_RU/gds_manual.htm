Справочное руководство по GDS

В кратком справочном руководстве по Global Diagnostic System (GDS) приводится обзор GDS.

Вся информация, содержащаяся данном руководстве, действительна на момент его публикации. Разработчики оставляют за собой право внесения любых изменений без уведомления об этом.

Общие сведения

GDS предназначена для использования специально обученным персоналом в целях диагностики и ремонта электронных систем автомобиля.  Было сделано все возможное для обеспечения их наиболее полной и точной технической информацией, основанной на доступных на момент публикации заводских данных.  Однако, разработчики оставляют за собой право внесения любых изменений без уведомления об этом.

В целях ознакомления с GDS, ее возможностями и приемами работы с нею, перед началом работы полностью прочтите руководство пользователя GDS.

GDS обладает следующими возможностями:

  • Чтение диагностических кодов неисправности
  • Чтение идентификационной информации о ECU
  • Чтение и графическое отображение параметров списков данных
  • Запись и хранение данных моментальных снимков
  • Функции управления
  • Функции настройки и сброса
  • Запись и хранение протоколов диагностики
  • Хранение информации по сеансам диагностики для каждого автомобиля
Важно:
GDS не поддеживает SPS программирование контроллеров автомобилей. Для программирования систем доступа SPS при помощи MDI следует использовать приложение TIS2Web / GlobalTIS SPS.

3. Установка GDS

Обзор компонентов GDS:

Программное обеспечение GDS состоит из следующих модулей, которые устанавливаются на компьютер в начале работы, а затем обновляются через определенные интервалы времени. [I]

  • GDS Base System
    Установленное на ПК базовое программное обеспечение GDS
  • Дополнительные модули GDS:
    Пакеты диагностических данных для различных моделей автомобилей, которые отвечают за выполнение диагностики контроллеров автомобилей.
  • GDS Smart Start:
    Агент приложений, который отвечает за сравнение версий GDS Base System и дополнительных модулей GDS, а также серверного программного обеспечения MDI Manager, установленных на TIS2Web/GlobalTIS и локальном компьютере. При наличии обновлений пользователь оповещается о необходимости их загрузки.
  • Программное обеспечение MDI Manager (отдельное приложение):
    Приложение для обновления и настройки устройств MDI (множественный диагностический интерфейс).


[I]

Установка:

Установка GDS может быть выполнена как с отдельного DVD, так и при помощи TIS2Web / GlobalTIS.

Установка с DVD GDS:

  • Необходимо запустить компьютер в режиме администрирования
  • Запустите программу установки из стартового меню DVD
  • Выберите язык программы установки
  • В случае если GDS ранее устанавливалась на данный компьютер, настройки и данные GDS, оставшиеся от этой установки, могут быть использованы повторно.
  • Примите требования лицензионного соглашения (необходимое условие для продолжения)
  • Выберите каталог для установки GDS
  • Начинается установка GDS
  • Программное обеспечение MDI Manager будет установлено в том случае, если оно отсутствует на данном ПК, либо на нем установлена более старая версия MDI Manager. При первой установке MDI Manager требуется, чтобы к компьютеру было подключено аппаратное обеспечение MDI. Более подробная информация приведена в руководстве по MDI.
  • Запуск GDS может осуществляться посредством пиктограммы на рабочем столе или при помоши менб Пуск/Программы/GDS после завершения установки. 
  • При необходимости настройте брандмауэр таким образом, чтобы GDS могла обмениваться информацией с устройством MDI.
  • Запускается приложение GDS Smart Start, и на дисплее отображается ход обновления программного обеспечения. После этого GDS запускается.

Установка при помощи TIS2Web / GlobalTIS:


[II]

Важно: Во время установки при помощи TIS2Web через Интернет загружается приблизительно 140 МБ данных.
  • ПК должен работать в режиме администрирования.
  • Запустите GDS с главной страницы TIS2Web / GlobalTIS [II]
  • При этом осуществляется загрузка GDS Smart Start с сервера TIS2Web. В зависимости от скорости соединения с Интернетом на это может потребоваться некоторое время.
  • Выберите каталог для установки GDS.
  • Введите пароль для доступа к Интернету (только при работе через прокси-сервер).
  • При необходимости настройте брандмауэр таким образом, чтобы GDS могла обмениваться информацией с устройством MDI.
  • Начинается установка GDS Smart Start. На это может уйти некоторое время.
  • Запустите GDS Smart Start. Происходит обновление программного обеспечения GDS. [III]
  • После завершения загрузки установочных файлов на дисплей выводится версия GDS и MDI Manager. Загрузка программного обеспечения начинается после подтверждения в соответствующем диалоговом окне. При нажатии на клавишу Показать процесс загрузки на дисплее отображается строка состояния, которая показывает ход загрузки.
  • Установка программного обеспечения MDI Manager осуществляется, если MDI Manager не установлено на данном компьютере, либо если установлена более старая версия MDI Manager. При первой установке MDI Manager требуется, чтобы к компьютеру было подключено аппаратное обеспечение MDI. Более подробная информация приведена в руководстве по MDI.
[III] [IV]

Обновление GDS посредством Smart Start:

Запустите глобальную службу каталога (GDS) или из TIS2Web/GlobalTIS или в качестве отдельной прикладной программы, используя меню 'Пуск' Windows. Перед загрузкой GDS начнет работу служба интеллектуального запуска, которая сравнит следующие версии установки GDS на ПК с контентом сервера TIS2Web / GlobalTIS:

  • GDS Smart Start
  • GDS Base System
  • MDI Manager

Smart Start сообщает о новых обновлениях, доступных для загрузки, если таковые имеются на сервере. [IV]

 
Важно: При выполнении обновления ПК должен работать в режиме администрирования.

Возврат к старой версии Base System GDS.

В случае если после обновления GDS в работе системы возникают проблемы, может быть выполнен возврат GDS к предыдущей версии.

Для этого необходимо запустить программу START ->PROGRAMS -> GDS -> GDS ROLLBACK. Обязательным требованием такого возврата является наличие на ПК установленной предыдущей версии GDS.

Система проверки лицензии GDS (аренда)

Система управления лицензиями GDS позволяет предотвратить неавторизованное использование системы GDS. Для запуска GDS из TIS2Web или GlobalTIS необходимо возобновить аренду GDS. Аренду GDS нужно возобновить через 7 дней в том случае, если прикладная программа запускалась за это время только в автономном режиме. 
Если аренда GDS истекла, в сообщении об ошибке пользователю дается подсказка о необходимости запустить программу GDS из TIS2Web/GlobalTIS, чтобы возобновить аренду.

Загрузка и обновление дополнительных модулей GDS

Дополнительные модули GDS представляют собой диагностические приложения, которые управляют обменом информации между GDS и контроллером автомобиля во время диагностики. Дополнительные модули GDS выпускаются для каждой модели автомобиля и постоянно обновляются через TIS2Web / GlobalTIS.

GDS Smart Start автоматически сообщает о наличии обновлений на TIS2Web / GlobalTIS после запуска процесса обновления.. Новые дополнительные модули сначала необходимо загрузить в локальный каталог на компьютере, после чего они могут быть установлены в GDS. Обновления уже установленных дополнительных модулей GDS загружаются автоматически. Требуется только установить их в главную базу данных.


Начальная загрузка и обновление дополнительных модулей GDS

  • Запустите глобальную службу каталога (GDS) или из TIS2Web/GlobalTIS или в качестве отдельной прикладной программы, используя меню 'Пуск' Windows.
  • Нажмите на кнопку Добавить новый диагностический пакет на экране входа в систему GDS
  • Нажмите на кнопку Обновить список дополнительных модулей на сервере. [V-1]. Этим символом обозначаются пакеты, имеющиеся на сервере, но еще не загруженные в систему. 
  • Выделите и загрузите необходимые дополнительные пакеты [V-2]. Функция фильтра помогает найти тот или иной дополнительный модуль Загруженные дополнительные модули обозначаются следующим символом. 
  • Установите загруженные пакеты в главную базу данных GDS. При этом установленные пакеты отображаются в правом окне приложения. [V-3].

Обновление дополнительных модулей GDS, установленных в главную базу данных системы:

  • Нажмите на кнопку Обновить диагностический пакет на экране входа в систему GDS [VI-1]. GDS сравнивает версии установленных дополнительных модулей с версиями, имеющимися на сервере, а также версиями уже загруженных обновлений GDS.
  • Загруженные пакеты отображаются в списке пакетов, готовых к установке, в нижнем левом окне. Чтобы начать установку, нажмите на кнопку Установить или Установить все. [VI-2].

Возврат к старым версиям дополнительных модулей

Процесс возврата к старым версиям дополнительных модулей в целом совпадает с процессом, описанным в разделе Начальная загрузка и обновление дополнительных модулей GDS.

  • Нажмите на кнопку Добавить новый диагностический пакет на экране входа в систему GDS
  • Нажмите на кнопку Обновить список дополнительных модулей на сервере. [V-1]
  • Выделите и загрузите необходимые старые версии дополнительных модулей. Функция фильтра помогает найти тот или иной дополнительный модуль
  • Установите загруженные пакеты в главную базу данных GDS.

Удаление GDS:

Для удаления GDS следует запустить следующую программу:

ПУСК -> ПРОГРАММЫ -> GDS ->GDS DE-INSTALLATION

Изменение языка GDS / Региональные настройки:

GDS работает с языковыми настройками операционной системы. Меню для изменения языка в GDS не придусмотрено. Для изменения языка GDS выполните следующие действия.

  • Откройте панель управления Windows в следующей последовательности: ПУСК -> НАСТРОЙКИ -> ПАНЕЛЬ УПРАВЛЕНИЯ
  • Откройте меню РЕГИОНАЛЬНЫЕ И ЯЗЫКОВЫЕ НАСТРОЙКИ
  • Выберите нужную страну в СТАНДАРТЫ И ФОРМАТЫ
  • Откройте РАСШИРЕННОЕ меню. Выберите нужный язык из списка ПРОГРАММЫ, НЕ ПОДДЕРЖИВАЮЩИЕ ЮНИКОД
  • Для того чтобы изменить единицы измерения, в которых отображаются полученные GDS значения, настройте соответствующим образом РЕГИОНАЛЬНЫЕ СТАНДАРТЫ.

4. приступая к работе

ПРИМЕЧАНИЯ:
Перед работой с GDS важно принять следующие меры предосторожности:

  • Использование устаревшей версии программного обеспечения может привести к поломке узлов автомобиля. Необходимо обновить GDS и множественный диагностический интерфейс до последней версии программного обеспечения.
  • Убедитесь в том, что аккумуляторная батарея автомобиля полностью заряжена. Напряжение аккумуляторной батареи по показаниям GDS должно быть от 12 до 14 В.
  • Однако во время работы с MDI зарядное устройство должно быть отключено от автомобиля.
  • Убедитесь в том, что контакты кабелей закреплены. Отсоединение кабеля может привести к выходу контроллера из строя.
  • При использовании для GDS портативного компьютера или другого устройства отображения информации (PDA и т.д.), убедитесь, что они надлежащим образом подключены к источнику питания. Если при работе от сети переменного тока произойдет отключение шнура питания, процесс программирования будет прерван, и блок управления может выйти из строя. Если портативный компьютер, на котором установлена GDS, работает от батарей, убедитесь в том, что они надлежащим образом заряжены.
ПРИМЕЧАНИЯ
Если требуется перепрограммировать ECU, необходимо, чтобы до начала работы он был установлен на автомобиль. Убедитесь в том, что аккумуляторная батарея автомобиля полностью заряжена.
Важно
Для того чтобы снизить риск потери сигнала, MDI, настройки обмена информацией должны соответствовать наиболее стабильному в условиях вашей мастерсой варианту.Возможен выбор между USB, Ethernet или Wireless Ethernet.

Последовательность включения GDS:

GDS может быть запущена из TIS2Web/GlobalTIS нажатием на пиктограмму GDS. Кроме того, GDS может быть запущена в независимом режиме при помощи стартового меню Windows.

  1. Для запуска GDS необходимо нажать на соответствующую пиктограмму главного меню TIS2Web/GlobalTIS. Для запуска GDS в независимом режиме, воспользуйтесь ярлыком ПУСК/ПРОГРАММЫ/GDS
  2. Запустите GDS Smart Start. При этом происходит обновление программного обеспечения. Если на сервере имеется новая версия GDS или MDI Manager, на дисплей будет выведено сообщение с предложением загрузить и установить соответствующую версию на компьютер. Для того чтобы начать загрузку и установку, нажмите на кнопку Продолжить, [IV]. Если GDS запущена в независимом режиме, обновление не производится.
  3. Загружается страница входа в систему GDS

Воспроизведение моментального снимка

Функция Воспроизведение моментальных снимков позволяет просматривать ранее записанные моментальные снимки данных автомобиля. Эта информация может быть использована для анализа проблем автомобиля, а также для определения триггеров, срабатывающих при обнаружении ошибок. Графическое отображение может использоваться для сравнения параметров в целях определения правильности работы компонентов. Для выбора записанных моментальных снимков можно использовать один из трех следующих методов:

  • Выбрать файл моментального снимка на Главная страница автомобиля и нажать на кнопку Обзор.
  • Дважды щелкнуть по выбранному моментальному снимку на Главная страница автомобиля.
  • Выбрать пункты меню Сервис, Меню, Воспроизведение моментальных снимков. Затем окрыть Менеджер моментальных снимков. Дважды щелкните по файлу моментального снимка, предназначенного к просмотру, или щелкните по списку Загрузить моментальный снимок и выберите в нем файл моментального снимка.

Независимо от того, какой метод используется для выбора, на экране Воспроизведение моментальных снимков появляется всплывающее окно Файл загружен. Нажмите на кнопку OK в этом окне.

Функция воспроизведения моментальных снимков [XIX-1]

Для воспроизведения моментальных снимков предусмотрено несколько функций.

Функция Воспроизведение используется для воспроизведения снимков. После нажатия на кнопку начинается воспроизведение снимка в прямом направлении.   При этом поперек экрана перемещается маркер.   Все остальные активные кнопки на этой панели инструментов становятся активными и отображаются серым цветом.
Функция Пауза используется для приостановки воспроизведения моментального снимка. При повторном нажатии на эту кнопку воспроизведение снимка возобновляется с места останова.
Функция Останов используется для прекращения воспроизведения моментального снимка. При этом, при повторном нажатии на кнопку Воспроизведение, воспроизведение моментального снимка начинается с начала.

Управление по времени [XIX-2]

Управление по времени используется для управления воспроизведением моментальных снимков. Эта фунция позволяет выбрать определенный кадр времени, отображать различные виды и управлять скоростью отображения. Следующие функции доступны непосредственно на панели инструментов Управление по времени.

Функция Закладка позволяет добавить специальный комментарий к текущему положению на дисплее и вернуться к этому положению в будущем. При нажатии на эту кнопку на дисплей выводится всплывающее окно. Введите имя закладки и нажмите на кнопку OK.
Фукнция Блокнот позволяет добалять комментарии в файл моментального снимка с привязкой к конкретному моменту времени. При нажатии на эту кнопку на дисплей выводится всплывающее окно. Введите комментарий к файлу моментального снимка и нажмите на кнопку OK.
Функция Загрузить моментальный снимок (синяя папка) позволяет открыть файл моментального снимка и подготовить его для последующего анализа. Если вы хотите просмотреть экспортируемый моментальный снимок, сначала распакуйте соответствующий архив *.zip.  Доступ к этой функции можно получить также щелчком по имеющемуся файлу моментальных снимков в окне Менеджер моментальных снимков.
Функция Стрелка вниз позволяет отобразить все опции Управление по времени. Отдельные опции доступны для работы в любой момент времени.
Функция Скорость отображения моментальных снимков позволяет управлять соответствующим параметром.   Можно выбрать одну из замедленных скоростей: 1/8Х, 1/4Х или 1/2Х.   Для воспроизведения моментального снимка в реальном времени выберите скорость 1.   Кроме того, можно выбрать одну из увеличенных скоростей: 2Х, 4Х и 8Х.

Выбор вида [XIX-3]

Каждый возможный вариант позволяет различным образом отображать данные автомобиля для анализа. Следующие функции доступны на панели инструментов Доступные окна.

  • Окно списка параметров - отображается входная информация из файла моментального снимка. Текущие значения параметров отображается на экране.
Важно
При отсутствии перевода имя параметра отображается на языке оригинала, который использовался при сборе информации.
  • Окно журнала - отображается вся информация из файла моментальных снимков Все значения параметров перечисляются в порядке их записи. Имеется возможность добавить или удалить отдельные параметры в список просмотра. Для этого следует щелкнуть по нужному параметру.
  • Окно состояния - отображаются значения всех параметров в тот или иной момент времени. Эти значения могут использоваться для сравнения со значениями, отображаемыми в окне триггеров.
  • Окно триггеров - отображается список всех системных и определенных пользователем триггеров Эта функция может использоваться, когда для диагностики неисправности важно сочетание тех или иных показателей.  Если условие триггера выполняется, на дисплее отображается треугольный значок предупреждения.
    • Загрузить триггеры - Функция позволяет загрузить файл, в котором содержатся определения триггеров.
    • Сохранить триггеры - Функция позволяет сохранить определения триггеров на жестком диске компьютера.
    • Удалить триггер - Функция позволяет удалить текущий триггер. После этого триггер больше не будет использоваться.
    • Редактировать триггер - функция позволяет вывести на дисплей всплывающее окно, при помощи которого можно отредактировать граничные условия и значения параметров, определяющие триггер.
    • Добавить триггер - позволяет отобразить всплывающее окно для настройки триггера.
  • Окно DTC - После начала воспроизведения эта функция позволяет отобразить на дисплее список записанных DTC.
  • Окно линейного графика - позволяет отображать данные графически в зависимости от времени. Выбрать параметры для отображения в этом окне можно, установив флажок Выбран у соответствующего параметра. В нижней части графика имеются четыре кнопки Увеличение (увеличить размер), Уменьшение (уменьшить размер), Закладки и Приостановить прокрутку. Каждая из этих функций может быть задействована нажатием на соответствующую кнопку.

7. ПОИСК И УСТРАНЕНИЕ НЕИСПРАВНОСТЕЙ

Проблемы с подключением MDI

Если разъемы MDI не закреплены или неправильно подключены к автомобилю/компьютеру, на дисплей выводится всплывающее окно с соответсвующим сообщением. Выполните следующие действия:

  • Убедитесь в том, что программное обеспечение Manager MDI не запущено одновременно с GDS.
  • Убедитесь в том, что MDI правильно подключен к автомобилю и компьютеру.
  • Воспользуйтесь заведомо "исправным USB-кабелем", снятым с другого устройства MDI, если имеется. Выполните все необходимые подключения и попробуйте установить соединение еще раз.
  • Воспользуйтесь заведомо "исправным соединительным кабелем DLC устройства MDI", снятым с другого MDI, если таковой имеется. Выполните все необходимые подключения и попробуйте установить соединение еще раз.
  • Если проблему устранить не удалось, обратитесь в Customer Support.
  • Убедитесь в том, что связь между GDS и MDI не блокируется брандмауэром.

Конфликт программного обеспечения GDS/MDI Manager

Программное обеспечение GDS и MDI Manager не могут работать одновременно. Если оба приложения запущены одновременно, между ними может имметь место конфликт. Более подробная информация об это ошибке приводится в разделе Детали. Если проблему устранить не удается, вы можете обратиться за соответствующей информацией в службу Customer Support.

Если конфликт программного обеспечения устранить не удается, и имеют место критические ошибки, на дисплей выводится всплывающее окно MDI Manger с сообщением об этом. Завершите работу программного обеспечения MDI Manger и вернитесь к работе с GDS. Если после введения GDS проблема сохраняется, обратитесь в Customer Support.
