General Information

The GDS is designed for use by trained service personnel to diagnose and repair automotive electronic systems.  Every attempt has been made to provide complete and accurate technical information based on factory service information available at the time of publication.  However, the right is reserved to make changes at any time without notice.

To familiarize with GDS and its capability, and how to use it, please read through the User's Guide before putting the GDS to work.

GDS provides the following capabilities:

  • Read out Diagnostic Trouble Code
  • Read out ECU identification information
  • Read out and graphical display of Data List Parameter
  • Record and store snapshot data
  • Control Functions
  • Configuration and Reset Functions
  • Record and store diagnostic protocol
  • Storage of diagnostic sessions per vehicle
Important:
SPS programming of vehicle controllers is not supported in GDS. For SPS Pass Through programming with the MDI use the TIS2Web / GlobalTIS SPS application.

3. GDS Installation

GDS Component Overview:

The GDS software contains the following modules which will be initially installed and updated frequently in certain periods. [I]

  • GDS Base System
    Installed GDS base software on the PC
  • GDS Deliverables:
    Diagnostic data packages for vehicle models which executes the diagnostic routines with the vehicle controllers.
  • GDS Smart Start:
    Application Agent which compares the versions of GDS Base System, GDS Deliverables and MDI Manager software Server between TIS2Web/GlobalTIS and the local installation. In case new updates are available they are prompted for download.
  • MDI Manager Software (separate Application):
    Application for update and configuration of the MDI Device (Multiple Diagnostic Interface)


[I]

Installation:

GDS can be installed from a separate installation DVD or directly as a download from TIS2Web / GlobalTIS.

Installation from GDS DVD:

  • PC must be started in Admin mode
  • Launch installation executable in the DVD start menu
  • Select the language for the install agent
  • Existing GDS configuration and data can be re-used in case a previous GDS installation is detected on the PC.
  • Accept the license agreement (required to proceed)
  • Select the installation directory for GDS
  • GDS Installation process starts
  • The MDI Manager software will be installed in case there is no or an older MDI Manager software installed on the PC. For the initial MDI Manager software installation, the MDI hardware needs to be connected during the installation process. For more details refer to the MDI manual.
  • GDS can be launched by the desktop symbol or via Start/Programs/GDS after completion of the installation process. 
  • If required configure the firewall to allow GDS communication with the MDI device.
  • GDS Smart Start starts, the displayed update process will be executed before GDS is launched.

Installation from TIS2Web / GlobalTIS:


[II]

Important: During installation from TIS2Web approximately 140 MB of data is downloaded via the internet.
  • PC must run in Admin mode
  • Start GDS in TIS2Web / GlobalTIS Home Page [II]
  • GDS Smart Start is downloaded from the TIS2Web server, depending of the internet connectivity this may take a moment.
  • Select the installation directory for GDS.
  • Enter the Internet Password (only if a proxy server is used for internet access)
  • If required configure the firewall to allow GDS communication with the MDI device.
  • When the GDS Smart Start installation begins, it may take a moment.
  • Launch of GDS Smart Start, the GDS update process will be executed. [III]
  • When the download of the installation files is finished, the GDS Version and the MDI Manager software version is displayed and can be downloaded by accepting the prompted dialogue. Via the button Show Download Progress the status bar of the update progress can be displayed.
  • The MDI Manager software will be installed in case there isn't an MDI Manager software installed on the PC or if an older version of MDI Manager software exists. For the initial MDI Manager software installation the MDI hardware needs to be connected during the installation process. For more details refer to the MDI manual.
[III] [IV]

GDS Update via Smart Start:

Launch GDS either from TIS2Web/GlobalTIS or as standalone application via the Windows start menu. Before GDS is loaded, the Smart Start agent is launched and compares the following versions of the PC GDS installation with the TIS2Web / GlobalTIS Server content:

  • GDS Smart Start
  • GDS Base System
  • MDI Manager

Smart Start prompts new updates to the user for download, if new versions are available on the server. [IV]

 
Important: The described update processes require that the PC is running always in Admin mode.

Rollback of GDS Base System Installation

The GDS installation can be rolled back to the previous installed version, in case a GDS update causes operational problems.

To execute the rollback function, go to START ->PROGRAMS -> GDS -> GDS ROLLBACK. A prerequisite of the rollback function is that an older GDS version must be installed on the PC.

GDS License Control System (Lease)

The GDS lease prevents the unauthorized use of GDS. To renew the GDS lease it is required to launch GDS from TIS2Web or GlobalTIS. The GDS lease requires renewal after 7 days if the application was launched during that time in standalone mode only. 
If the GDS lease is expired, an error message prompts the user to launch the GDS application from TIS2Web/GlobalTIS for lease renewal.

Download and Update GDS Deliverables

GDS Deliverables are diagnostic applications, which control the communication and diagnostic services between GDS and the vehicle controller. GDS deliverables are released per model and will be permanently updated on TIS2Web / GlobalTIS.

GDS Smart Start identifies automatically if new or updated deliverable versions are available on TIS2Web / GlobalTIS when the update process is triggered. New deliverables need to be downloaded into a local directory on the PC at first before they can be installed in GDS. Updates of already installed deliverables will be automatically downloaded and just need to be installed in the GDS Master Database.


Initial Download and Update of GDS Deliverables

  • Launch GDS either from TIS2Web/GlobalTIS or as standalone application via the Windows start menu.
  • Select the button Add New Diagnostic Package from the GDS Login Screen.
  • Press the button Update Deliverable list from Server. [V-1]. On the server available but not yet downloaded packages are displayed with the symbol. 
  • Mark and download the deliverable packages [V-2]. The filter function helps to search for specific deliverables. Downloaded deliverable packages are displayed with the symbol. 
  • Install the downloaded packages into the GDS Master Database; installed packages are displayed in the right application window [V-3].


[V]

Update Installed GDS Deliverables in Master Database:

  • Click on the button Update Diagnostic Package in the GDS Login screen [VI-1]. GDS compares the versions of the installed GDS deliverables with the deliverables available on the server and downloads updated packages.
  • The downloaded packages are displayed for installation in the lower left window. Select the button Install or Install all to begin with the installation. [VI-2].


[VI]:

Rollback Deliverable Update

The rollback process is in general the same process as described in chapter Initial Download and Update of GDS Deliverables

  • Select the button Add New Diagnostic Package from the GDS Login Screen.
  • Press the button Update Deliverable list from Server. [V-1]
  • Mark and download the desired older deliverable package versions. The filter function helps to search for specific deliverables.
  • Install the downloaded packages into the GDS Master Database.

4. Getting Started

NOTICE:
Prior to performing GDS, it is important to observe the following precautions:

  • Using an outdated version could damage vehicle modules. The GDS and Multiple Diagnostic Interface must have the latest software.
  • Make sure the vehicle battery is fully charged Battery voltage for GDS should be between 12 and 14 volts.
  • However, a battery charger must not be connected to the vehicle when using the MDI.
  • Make sure the cable connections are secured. A disconnected cable MAY cause controller failure.
  • In using a laptop computer/other display device (PDA etc.) for GDS, ensure that the power supply is properly connected. If powered by AC and the power cord becomes disconnected, it could interrupt programming and cause damage to the control module. If the laptop is operating from its internal power source (batteries), then make sure it is adequately charged to complete the GDS process.
NOTICE
If an ECU is to be programmed it must be installed in the vehicle before beginning this process. Make sure the vehicle battery is fully charged.
Important
In order to reduce the potential for signal loss, the MDI should be configured for the most stable communication option at your location.You can choose from USB, Ethernet, or Wireless Ethernet.

GDS Start - Up Procedure:

GDS can be started through TIS2Web/GlobalTIS by clicking to the GDS icon. Alternatively GDS can be launched in standalone mode through the Windows Start menu.

  1. To start click on the GDS symbol from TIS2Web/GlobalTIS main menu. If starting GDS in standalone mode, click on START/PROGRAMS/GDS
  2. Launch of GDS Smart Start, the displayed update process will be executed. If a new GDS or MDI Manager version is available on the server it will be prompted for download and installation for the PC. Click Continue to start the download or installation [IV]. If GDS is launched in standalone mode no update is possible.
  3. GDS Login Page is loaded.

Snapshot Replay

The Snapshot Replay function allows reviews of previously recorded vehicle snapshot data. This information can be used in analysis of vehicle problems and detection of trigger error conditions. The graph function can be used to compare parameters to see if a component is functioning properly. Recorded snapshots can be selected for viewing by the following three methods:

  • Selecting a snapshot file from the Vehicle Home Page and clicking on the Review selection.
  • Double clicking on the selected snapshot from the Vehicle Home Page.
  • Entering the Tools selection from the Menu selection and selecting Snapshot replay. Next, open the Snapshot Manager. Double click on the snapshot file you wish to view or click on the Load Snapshot selection and select a snapshot file.

Regardless of selection method, a File Loaded pop-up window will be displayed from the Snapshot Replay screen. Select OK from the pop-up window.

Snapshot Play Functions [XIX-1]

There are several snapshot play functions that are used to replay recorded snapshots

The Play function is used to play the snapshot forward by clicking on this selection.   A marker will move across the screen when this selection is used.   All selections in this area turn grey when they are active.
The Pause function is used to pause play of the snapshot. When this selection is clicked a second time the snapshot will start playing where it left off.
The Stop function is used to stop playing the snapshot. The snapshot will start from the beginning, if Play is selected after using the Stop function.

Time Control [XIX-2]

Time Control is used to control the snapshot replay. This function allows selection of a specific time frame, display of different views, and control of display speed. The following functions can be selected directly under Time Control.

The Bookmark selection allows you to add a specific comment in the current display and return to this point in the future. When this option is selected, a pop-up window is displayed. Enter a bookmark name and select OK.
The Notepad can be selected to add comments to the snapshot replay file at the current point of time. When this option is selected, a pop-up window is displayed. Enter a comment for the snapshot file and select OK.
The Load Snapshot (blue folder) opens a snapshot file and makes it ready for further analysis. If you want to view an exported snapshot, first decompress the corresponding archive *.zip.  This function is also available by clicking on an available snapshot file from the Snapshot Manager.
The Down Arrow displays all Time Control options when selected. Individual options can also be selected by clicking on the item you want.
The Snapshot Display Speed can be controlled by selecting this option.   Slow speeds of 1/8X, 1/4X and 1/2X can be selected.   The real time speed can also be selected at 1.   Faster speeds of 2X, 4X, and 8X can also be obtained by using this selection.

7. TROUBLESHOOTING

MDI Connection Problems

If the MDl connections are loose or not properly connected to the vehicle/computer, a pop-up window will be displayed. Perform the following steps:

  • Make sure the MDl Manager software is not running at the same time GDS is being performed.
  • Make sure the MDl is properly connected to the vehicle and computer
  • Use a known "good USB cable" from another MDI unit, if available. Perform all connections and try again.
  • Use a known "good MDI DLC connection cable" from another MDI unit, if available. Perform all connections and try again.
  • If the condition is still present, contact Customer Support.
  • Make sure that the firewall does not block GDS/MDI communication.

GDS/MDI Manager Software Conflict

GDS software and MDI Manager software cannot be running at the same time. If both software programs are run at the same time, a software conflict may result. The Details selection will provide possible information regarding the error. You may be asked by Customer Support to provide this information if the problem persists.

If the software conflict continues to occur, a "Fatal error has occurred" pop-up window may be displayed the MDI Manger software. Shutdown the MDI Manger software and go back into the GDS program. If the problem is still present after entering GDS, contact Customer Support.
