全球TIS安装的用户手册


1.  计划与背景

1.1 介绍

在如今欧宝合作伙伴的服务范围内，TIS2000应用程序是一个重要的信息来源。  不断更新的DVD被用来发送技术文件、测试设备和ECU（电子控制单元）的软件。

TIS2000正在重新修订，将作为新的应用程序通过互联网提供。  这样做的好处很明显：信息可以做到最新并避免安装问题。  朝互联网方向发展的一个中间步骤是拥有这个新的应用程序，它仍然以DVD为基础，并设计用于本地电脑的安装。  这种方法可以降低对互联网的依赖性。

此外，较早的TIS2000得以改进，压缩了信息种类，以便更快地访问特定的文件。  您现在将找到一份协调的内容目录，不仅能看到修理和拆除的说明，还有适用的现场修理、检查程序和电路图。

1.2  缺少的功能

新的应用程序还不是TIS2000的一个完整取代品。  在最初阶段，它将包含SPS数据库和Tech2及MDI（Tech2的后继者）的诊断软件更新。  在此阶段，用户应该对新的系统加以熟悉。

在2009年底的较晚阶段，来自TIS2000的其余信息，也就是维修信息、电路图、工作时间、检查程序以及所有其它文件将被整合在内。  同时，那将意味着目前的TIS2000的终结。

2.  用户手册

2.1  安装（服务器或单机电脑）

和TIS2000一样，此DVD需要安装在服务器或本地电脑（单机）上。  然而，在实际处理上还有一些不同，见下文所述。  另一个基本的不同点是需要注册。  这意味着全球TIS只能在用户成功证实自己的授权后才能使用。  有关详情，请参阅2.1.4。

2.1.1  安装前：

  • 想要执行一次安装或在以后进行一次更新的用户必须在电脑上具有管理者权限。
  • 在开始安装前，请确认您的Windows系统已用最新的服务程序包（Service Packs）更新过。
  • 全球TIS的安装目录不能存在，或者必须为空。  目录的名称不可以包含特殊字符。
  • 如果您的电脑上连接了一个USB型的硬件钥匙（HWK），在安装时，您必须拆除它。  否则Windows将会自动安装驱动器程序，而这会影响以后的操作。  然后安装的会是一个错误的驱动器。

2.1.2 安装时：

  • 系统将询问安装语言。 该选项指的只是安装程序所用的语言。 用户界面和文件的语言设定是通过别的方法定义的。
  • 系统将提示输入端口，例如“Transbase Server Port”。 除非您是特意要改变它们，否则不应该那么做。
  • 系统将提示输入一个主机名称。这在单机电脑上应该保留“localhost”不变，或者改成电脑在服务器上的名称（查看您桌面上的“my computer”，点击鼠标右键，选择电脑名称，选择按钮“change”来得到正确的电脑名称）。
  • 系统将提示输入一个“Mail server”。  此功能在您想直接从电脑上发送反馈意见而不经过打印传真时使用。  默认设置的值不需要更改。

2.1.3  安装后在首次启动前：

安装完成后，用户需要手动启动应用程序，通过“Start” - “Programs” - “GlobalTIS” - “GlobalTIS”来进行。  如果您有一个硬件钥匙（HWK），此时可以将它连接到电脑上。  这需要在第一次启动前进行。

 

2.1.4 注册：

只有经授权的用户才能用全球TIS进行工作。  授权通过一个注册过程或一个现有的HWK获得。  后一种情况是最简单快捷的方式，可以立即获得访问权。

2.1.4.1  HWK步骤：

在第一次启动全球TIS前

  • 连接HWK到单机电脑，或
  • 连接HWK到服务器电脑上（仅WIN2000和XP）

和TIS2000不同的是，HWK不可以连接到一台客户机电脑上。  它的序列号码将自动成为授权（许可密码）。  请注意，一个HWK只可以用来获取一个许可密码。  不过，它将保持其TIS2000的可操作性，即使在转换之后。

2.1.4.2  Window2003服务器：

HWK在一台带Windows2003的服务器电脑上无法进行转换。  它需要执行在线注册或用传真手工注册（见下文）。

2.1.4.3  软件钥匙程序：

在没有HWK可用的情况下，应用程序启动后，用户会被自动引导进行一个注册程序。  每个标记了*）的输入栏都必须用要求的数据填写。  在表格末尾，用户可以选择是进行在线注册还是手工注册。  进行在线注册时，用户相关的数据被电子传输到一个中心，许可密码在那里生成并发回。  这个许可密码是一个编码号码，必须在“license key”栏中输入。

进行手工注册时，系统将自动生成一份PDF格式的文件。  它可以作为电子邮件的附件发送到提供的地址，也可以通过传真发送。  然而，最好是采用电子邮件，因为其处理会快捷许多。

2.1.5  销售品牌的订阅

在注册过程中，用户将得到一份订阅，严格用于1个销售品牌。  对于每个额外的品牌，其注册需要通过再次运行注册程序来扩展。  订阅延长后，全球TIS需要关闭并再次启动，以显示新的应用程序图标。

2.1.6  订阅修改

如果有必要进行订阅人地址的修改，这必须通过传真发送。  目前来说，这个操作无法用在线程序进行。  因此，如果需要订阅额外的品牌，现有的注册必须加以修改。  其方法是，启动全球TIS并选择“Admin”标签。  在那里您可以选择一个新的订阅，按下手工或在线注册的按钮。

如果您没有足够的用户授权，则需要同样的程序来增加用户数量。  该数字限制了同时连接到一个服务器上的用户数量。  在大多数情况下，这个数字不需要修改。

2.1.7  更新全球TIS

新版本的全球TIS在DVD上发布时，它代表了现有安装的一次更新。  将DVD放入DVD驱动器。  在电脑的“Autoplay”选项启用时，更新程序将自动启动。  如果此功能在您的系统中没有启用，您需要手动从GlobalTIS DVD的根目录启动setup.exe。  GlobalTIS不会自动搜索DVD驱动器上的更新。

2.1.8  应用程序的启动

应用程序可以通过点击“Start”菜单中的条目启动。  默认的条目可以如下找到“Start” - “Programs” - “GlobalTIS” - “GlobalTIS”

2.1.8.1  启动客户机

如果应用程序要在一台客户机电脑上启动，那么您所需要的只是“Internet Explorer”。  在地址栏中必须输入服务器的名称，例如 http://server-name:port/tis2web.

例如： http://asterix:9080/tis2web

典型“端口”为“9080”，或者是定义了“Tomcat”端口时在安装过程中输入的值。
此地址应该作为书签保存在您浏览器的收藏夹中。  GlobalTIS只应该和Microsoft Internet Explorer一同使用。

2.1.8.2 启动服务器

您启动一台服务器时，会需要用一个用户名登录。  在第一次启动时，您必须输入自己选择的一个新用户。  一旦您再次启动，您可以选择这个用户或者输入一个新的用户。

2.1.9  Java实时环境

需要它的模块有

  • 软件下载
  • 维修编程系统（SPS）
  • 安全进入编程设定
  • 快照
  • Tech2 View
  • Techline打印

如果电脑已经安装了一个高于1.4x版本的Java，那么Java应用程序应该在GlobalTIS模块启动时自动被下载。  这只会发生一次，或者在服务器上有修订版本时发生。
如果还没有安装Java，或者版本较老，则您需要先进行安装。  版本的Java可以从“Help”以及此部分进行下载： 一般说明 -  下载应用程序

2.1.10  Java代理服务器设置

如果Java应用程序不启动，则可能需要更改“Java Web Start”的配置。  代理服务器的设定需要停用。
“File” - “Preferences” - “General”。  在代理服务器处选择“None”

2.2  屏幕设定

要支持有效方便的软件处理，优化屏幕设定极为重要。  一般来说，有2部分需要加以注意：

设定文件范围数值的适当大小。  选择图标“Settings”并更改参数“Display Area Height”，直到文件在整个页面上展开。

按下功能键“F11”，改动互联网浏览器的显示模式为“Full Screen”。

“Toggle display”按钮在两个显示模式之间进行转换，目录显示在文件内容旁，或只显示文件内容。  这有助于最佳利用可用的屏幕空间。  调用按钮在图标条上可以使用，如下所示：

2.3  语言设定

全球TIS的语言是由浏览器的设定所决定的。  要改变语言，可以打开浏览器的“Tools”选单，选择“Internet Options”以及“Languages...”按钮。  按照浏览器的说明进行操作。
要启用更改，用户需要再次登录应用程序。

2.4  浏览器设定

当前的版本最适用于屏幕分辨率高于1024 x 768像素，而文字大小为“smaller”（互联网浏览器“View” - “Text Size”）。  如果您决定使用其它的设定，一个滚动条会出现在屏幕边缘，您要查看整个文件时需要使用它。

2.5  访问管理

在TIS2000中有一个被成为“Session Management”的功能。  它允许数个用户在独立的环境中用TIS2000进行工作。  所有的设定如车辆类型、车型年和搜索标准都是得到保存的，其他用户可以不受干扰地找到自己的信息。

全球TIS不具有同样的技术，但是有一个相似的机制。  如果另一名用户需要用系统进行工作，但是不想重新设置所有以前定义的设定，他只需要用互联网浏览器启动另一个窗口并用自己的个人ID登录。

2.6  和TIS2000 - 的不同点

  • 操作更多地由图标控制。
  • 车辆品牌在TIS2Web安装后只需要定义一次。
  • 启动页面所提供的符号少于TIS2000。
车辆品牌在TIS2Web安装后只需要定义一次。  启动TIS2Web，点击右上角的图标“Settings”并选择可选条目中的一个。  它即使在程序被关闭后仍然会保留设定。

更多重要的建议可以在右上角的“Help”’功能中找到。


2.7  前进与后退按钮的功能

互联网浏览器的前进与后退按钮在全球TIS的操作期间不起作用。  因此，不要使用它们。  在图标条中有一个清楚的符号。

基本上来说，TIS2000已知的模块都被重新加以利用并整合在网络技术中。  因此我们认为您不再需要熟悉过程。

2.10 反馈

如果您在全球TIS的操作中发现了错误，有一种方便的方式可以向我们报告。  一个内置的反馈表已经填好了背景信息，您只需要完成一个专门的错误描述和您的联系数据。  您所要做的只是点击“Send”按钮，它将自动生成电子邮件发送给我们。  您不再需要创建一份单独的邮件或给我们发传真。

3. 卸载

如果您想从您的电脑上卸载全球TIS，有两种可能的方式：

  1. 一个专门的卸载程序可以通过“Start” - “Programs” - “GlobalTIS” - “Uninstall GlobalTIS”来启动
  2. 软件也可以通过“Start” - “Settings” - “Control Panel” - “Add or Remove Programs”来卸载。  您需要搜索到“GlobalTIS”的条目，并点击按钮“Change/Remove”

少量关于先前安装配置的信息将保留在系统中。  例如您在注册过程中输入和接收的信息。  它们可以重新使用，并使GlobalTIS的新安装更容易。  但是如果您在新GlobalTIS安装期间选择了“Ignore old settings”，所有注册数据将丢失，而您必须再次执行获取许可的程序。
