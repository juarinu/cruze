GlobalTIS 설치를 위한 사용자 설명서


1.  계획 및 배경

1.1 소개

현재 Opel 파트너의 정비 부분에서 TIS 2000 애플리케이션은 중요한 정보원입니다.  DVD의 지속적인 업데이트는 기술 문서, 테스트 장치 및 ECU(전자 제어 장치) 소프트웨어 등을 제공합니다.

TIS 2000은 재 작업 중이며 인터넷을 통해 새로운 애플리케이션으로 이용 가능하게 됩니다.  이로 인해 얻게 되는 명확한 이점은 최신 정보 및 설치 문제 방지입니다.  인터넷으로 전환되는 중간 절차는 이 새로운 애플리케이션을 보유하는 것이나, 여전히 DVD에 수록되어 있으며 지역적으로 설치하기 위해 설계되어 있다는 것입니다.  이와 같이 하면 인터넷 가용성의 의존도가 줄어듭니다.

추가적으로 이전 TIS 2000은 정보 범주를 축소하여 특정 문서를 보다 신속하게 접근할 수 있도록 하는 방향으로 변경되었습니다.  이젠, 잘 어우러진 목차를 확인할 수 있습니다. 수리 및 탈거 지침은 물론 해당 현장 해결책, 점검 절차 및 배선도로 확인 가능합니다.

1.2  포함되지 않은 기능

새로운 애플리케이션은 TIS 2000을 아직 완전히 교체한 것은 아닙니다.  초기 단계에서는 Tech 2 및 MDI(Tech 2 이후 버전)에 대한 SPS 데이터베이스 및 진단 소프트웨어 업데이트가 수록됩니다.  이 단계에서 사용자는 새로운 시스템에 친숙해져야 합니다.

2009년 연말까지 이후 단계에서는 TIS 2000의 잔여 정보(예: 서비스 정보, 배선도, 작업 시간, 점검 절차)와 기타 모든 문서가 통합될 예정입니다.  동시에, 현재 TIS 2000이 폐간된다는 의미도 됩니다.

2.  사용자 설명서

2.1  설치(서버 도는 독립용 PC)

TIS 2000과 같이 DVD가 서버나 로컬 PC(독립형)에 설치되어야 합니다.  그러나, 아래와 같이 취급 시 일부 차이점이 있습니다.  다른 기본 차이점은 등록을 해야 한다는 점입니다.  즉, GlobalTIS는 사용자가 승인을 받은 사용자라는 것을 성공적으로 확인한 후에만 사용 가능합니다.  자세한 사항은 2.1.4항을 참조하십시오.

2.1.1  설치 전:

  • 이후 설치 또는 업데이트하려는 사용자는 해당 PC에 관리자 권한을 보유해야 합니다.
  • 설치를 시작하기 전 Windows 시스템이 최신 서비스 팩으로 업데이트되었는지 확인해 주십시오.
  • GlobalTIS의 설치 디렉토리는 존재하지 않거나, 존재하더라도 안에 아무것도 없어야 합니다.  디렉토리의 이름에 특수 문자가 포함되면 안 됩니다.
  • 하드웨어 키(HWK) USB형이 PC에 연결되어 있는 경우, 설치 중 이를 빼내야 합니다.  그렇지 않은 경우, Windows가 자동으로 드라이버 소프트웨어를 설치하게 되어 추후 작동에 영향을 주게 됩니다.  잘못된 드라이버가 설치됩니다.

2.1.2 설치 중:

  • 시스템이 언어를 물어보게 됩니다. 이 선택은 설치 절차에만 관련이 있습니다. 사용자 인터페이스 및 문서의 언어 설정은 서로 다르게 규정됩니다.
  • 이 시스템은 포트 입력(예: '트랜스베이스 서버 포트')에 대해 알려줍니다. 의도적으로 변경할 생각이 없으면 이를 바꾸면 안 됩니다.
  • 시스템이 호스트명의 입력에 대해 알려줍니다.독립형 PC에서는 'localhost'로 놔두거나 서버에서는 컴퓨터 이름으로 변경해야 합니다. (올바른 컴퓨터 이름을 위해서 데스크톱의 '내 컴퓨터'를 오른쪽 마우스 버튼으로 클릭하고 컴퓨터 이름을 선택하여 '변경' 버튼을 선택합니다.)
  • 시스템이 '메일 서버'에 대해 알려줍니다.  이 기능은 팩스용 출력물을 실행하지 않고 이 PC에서 직접적으로 피드백을 전송하려고 할 때 사용됩니다.  기본 값에 따라 이 값은 변경할 필요가 없습니다.

2.1.3  설치 후이나 단 처음 시작하기 전입니다.

설치가 종료되면 '시작' - '프로그램' - 'GlobalTIS' - 'GlobalTIS'를 통해 직접 애플리케이션을 시작해야 합니다.  하드웨어 키(HWK)가 있는 경우, 이를 PC에 연결할 때입니다.  처음으로 시작하기 전에 실시해야 합니다.

 

2.1.4 등록:

오직 승인을 받은 사용자만이 GlobalTIS를 사용하여 작업할 수 있습니다.  승인은 등록 절차나 기존의 HWK를 통해 이루어집니다.  후자의 경우가 즉시 접근할 수 있는 가장 편리하고 신속한 방법입니다.

2.1.4.1  HWK 절차:

처음으로 GlobalTIS를 시작하기 전

  • HWK를 독립형 PC에 연결 또는
  • 서버 PC에 HWK를 연결(WIN2000 및 XP)

TIS 2000과는 달리 HWK는 클라이언트 PC에 부착하면 안 됩니다.  관련 일련 번호가 자동으로 승인됩니다(라이센스 키).  HWK는 라이센스 키를 확보한 후에만 한 번 사용 가능하다는 점을 유념하시기 바랍니다.  그러나, 전환된 후에도 TIS 2000에 계속 사용할 수 있습니다.

2.1.4.2  Windows 2003 서버:

Windows 2003를 통해 서버 PC에 HWK를 전환할 수 없습니다.  온라인 등록 또는 팩스를 통해 직접 등록해야 합니다(아래 참조).

2.1.4.3  소프트웨어 키 절차:

이용 가능한 HWK가 없는 경우, 애플리케이션이 시작된 후 사용자는 등록 절차를 통해 자동으로 안내받게 됩니다.  *)가 표시된 입력 란 각각 요구 데이터를 기입해야 합니다.  이 형식의 끝 부분에서 온라인 등록 또는 직접 등록 여부를 선택할 수 있습니다.  온라인 등록의 경우, 사용자 관련 데이터가 중앙 위치로 전자 전송됩니다. 이 위치에서 라이센스 키가 생성된 후 다시 전송됩니다.  이 라이센스 키는 코딩된 번호이며 '라이센스 키' 란에 입력되어야 합니다.

직접 등록의 경우, 시스템이 PDF 형식으로 문서를 자동 생성합니다.  이메일 첨부 파일 형태로 지정된 주소에 전송되며, 팩스로도 전송 가능합니다.  그러나, 보다 신속하게 처리되므로 이메일이 선호됩니다.

2.1.5  판매 가입신청

등록 절차 중, 사용자는 정확하게 1개의 판매 가입신청을 확보하게 됩니다.  추가 판매 각각의 경우 등록 과정을 다시 거쳐 등록을 연장시켜야 합니다.  신청이 연장된 후, 신규 애플리케이션 아이콘이 표시되도록 하기 위해서는 GlobalTIS를 닫고 다시 시작할 필요가 있습니다.

2.1.6  가입신청 변경

가입신청자의 주소와 관련하여 변경이 필요한 경우, 팩스를 통해 전송해야 합니다.  당분간 온라인 절차에서는 효과가 없습니다.  그러므로, 추가 판매 가입신청이 필요한 경우, 기존의 등록을 변경해야 합니다.  이렇게 하려면 GlobalTIS를 시작하여 '관리자' 탭을 선택합니다.  이 탭에서 새로운 가입신청을 선택하여 직접 등록 또는 온라인 등록 버튼을 누를 수 있습니다.

만약 권한 받은 사용자 라이센스가 부족하여 사용자 수를 늘려야 하면 같은 절차를 실행해야 합니다.  이 수는 서버에 연결된 동시 접속자 수로 제한됩니다.  대부분의 경우, 이 수를 변경할 필요가 없습니다.

2.1.7  GlobalTIS 업데이트

새로운 버전의 GlobalTIS가 DVD에 수록된 경우, 기존 설치 업데이트를 의미합니다.  DVD 드라이브에 DVD를 입력합니다.  업데이트 절차는 PC의 '자동 재생' 옵션을 이용할 수 있을 때 자동으로 시작됩니다.  이 기능이 시스템에 활성화되어있지 않으면, GlobalTIS DVD 루트 디렉터리에서 수동으로 setup.exe를 실행해야 합니다.  GlobalTIS는 DVD 드라이브에서 업데이트를 자동 검색하지 않습니다.

2.1.8  애플리케이션 시작

'시작' 메뉴의 입력을 클릭하면 애플리케이션을 시작할 수 있습니다.  기본 입력은 '시작' - '프로그램' - 'GlobalTIS' - 'GlobalTIS'에서 찾을 수 있습니다.

2.1.8.1  클라이언트를 시작합니다.

애플리케이션을 클라이언트 PC에서 시작해야 하는 경우, 오직 필요한 것은 '인터넷 익스플로러' 입니다.  주소란에서 서버 이름을 입력해야 합니다(예: http://server-name:port/tis2web).

예: http://asterix:9080/tis2web

'포트' 는 일반적으로 '9080' 이거나 'Tomcat' 포트가 규정되었을 때 설치 시 입력했던 값입니다.
주소는 브라우저 즐겨찾기의 북마크로 저장되어야 합니다.  GlobalTIS는 마이크로소프트 인터넷 익스플로러에서만 사용해야 합니다.

2.1.8.2 서버를 시작합니다.

서버를 시작하면 사용사명으로 로그인해야 합니다.  가장 처음에 시작할 때는 원하시는 사용자명을 입력해야 합니다.  다시 시작할 경우, 이 사용자를 선택하거나 새로운 사용자를 생성할 수 있습니다.

2.1.9  Java Runtime 환경

이는 다음과 같은 모듈에 필요합니다.

  • 소프트웨어 다운로드
  • 서비스 프로그래밍 시스템(SPS)
  • 보안 액세스
  • 스냅샷
  • Tech2 View
  • 테크라인(Techline) 인쇄

PC에 1.4x 이상의 Java 버전이 이미 설치되어 있는 경우, GlobalTIS 모듈이 시작되고 있을 때 Java 애플리케이션이 자동으로 다운로드됩니다.  이는 오직 한 번 또는 서버에 개정 버전을 이용 가능하게 된 후에 발생합니다.
아직 Java가 설치되지 않았거나 버전이 오래된 경우 먼저 설치해야 합니다.  Java 버전은 '도움말' 및 다음 항목에서 다운로드할 수 있습니다. 일반 -  애플리케이션 다운로드

2.1.10  Java 프록시 설정

Java 애플리케이션이 시작되지 않는 경우, 'Java 웹 시작' 애플리케이션 구성을 변경해야 할 수도 있습니다.  프록시 설정이 비활성화되어야 합니다.

2.2  화면 설정

소프트웨어를 효율적이고 쉽게 처리하는데 도움이 되도록 최적 화면 설정이 중요합니다.  기본적으로 2가지 사항에 주의해야 합니다.

문서 영역 값에 대한 해당 크기를 설정합니다.  '설정' 아이콘을 선택한 후 문서가 전체 페이지로 확자오딜 때까지 '디스플레이 영역 높이' 파라미터를 변경합니다.

기능 키 'F11'를 눌러 인터넷 익스플로러 디스플레이 모드를 '전체 화면'으로 변경합니다.

'토글 디스플레이' 버튼을 누르면 나란히 위치한 문서 옆의 목차 디스플레이 모드가 서로 바뀌거나, 문서의 내용만 바뀝니다.  이렇게 하면 이용 가능한 화면 공간을 최적으로 사용할 수 있습니다.  토글 버튼은 아래와 같이 아이콘 바에서 이용 가능합니다.

2.3  언어 설정

GlobalTIS의 언어는 브라우저 설정을 통해 결정됩니다.  언어를 변경하려면 '도구' 메뉴 브라우저를 열고 '인터넷 옵션' 을 선택한 후 '언어...' 버튼을 누릅니다.  브라우저 지침을 준수합니다.
변경 사항을 활성화시키려면 다시 애플리케이션에 로그온해야 합니다.

2.4  브라우저 설정

현재 버전은 1024 x 768 화소인 화면 해상도와 '작은' 텍스트 크기에 맞게 최적화되었습니다(인터넷 익스플로러 '보기' - '텍스트 크기').  다른 설정을 사용하려는 경우, 문서 전체를 확인할 때 사용해야 하는 화면 경계선에 스크롤 바가 나타날 수도 있습니다.

2.5  세션 관리

TIS 2000에는 '세션 관리' 라는 기능이 있습니다.  이 기능을 사용하면 여러 사용자가 TIS 2000으로 작업을 할 수 있습니다. 단, 서로 독립적인 환경에서만 가능합니다.  차량 형식, 연식 등의 모든 설정과 검색 기준이 바뀐 관계로 다른 사용자가 서로 지장을 주지 않고 정보를 검색할 수 있습니다.

GlobalTIS의 경우, 이와 똑같지는 않지만 유사한 방식으로 실행됩니다.  다른 사용자가 시스템 작업을 해야 하나 이전에 규정된 설정을 모두 재설정하려고 하는 경우, 인터넷 익스플로러를 사용하여 다른 창을 시작한 후 개인 ID로 로그인 하기만 하면 됩니다.

2.6  TIS 2000 과의 차이점

  • 작동은 아이콘으로 더 많이 제어됩니다.
  • 차량 제조는 TIS2Web을 설치한 후 오직 한 번만 규정되어야 합니다.
  • 시작 페이지는 TIS 2000에서 사용할 수 있는 기호가 더 적습니다.
차량 제조는 TIS2Web을 설치한 후 오직 한 번만 규정되어야 합니다.  TIS2Web을 시작한 후 상단 우측 모서리에 있는 '설정' 아이콘을 클릭한 후 이용 가능한 항목 중 하나를 선택합니다.  프로그램이 종료된 후에도 계속 설정된 상태가 됩니다.

더 중요한 제안 사항은 상단 우측 모서리에 있는 '도움말' 기능에서 찾을 수 있습니다.


2.7  앞으로 및 뒤로 버튼의 기능

인터넷 익스플로러의 앞으로 및 뒤로 버튼은 GlobalTIS 작동 시 작동하지 않습니다.  그러므로, 사용하면 안 됩니다.  대신, 아이콘 바에 명확한 기호가 있습니다.

 

2.8  소프트웨어 다운로드, 스냅샷 표시 및 SPS

본질적으로, 이미 알려져 있는 TIS 2000 모듈이 다시 사용되었으며 웹 기술에 통합되었으므로,  더 이상 이에 대해 학습할 필요는 없는 것으로 생각됩니다.

2.9 지원

소프트웨어 설치 및 작동 중 문제가 발생하는 경우, DAC(대리점 지원 센터)에 이러한 문제를 알려줍니다.

2.10 피드백

GlobalTIS 작동 시 오류를 감지한 경우, 볼보로 이를 보고할 수 있는 쉬운 방법이 있습니다.  동봉된 피드백 양식은 문맥 정보로 이미 기입된 상태이므로, 해당 오류 설명 및 문의 데이터만 작성하면 됩니다.  단지 '전송' 버튼을 클릭하기만 하면 볼보로 전송되는 이메일이 자동 생성됩니다.  개별 메일을 생성하거나 팩스로 전송할 필요가 없습니다.

3. 제거

PC에서 GlobalTIS를 제거하려는 경우, 이에 대한 방법은 2가지입니다.

  1. 전용 제거 절차는 '시작' - '프로그램' - 'GlobalTIS' - 'GlobalTIS'를 통해 시작 가능합니다.
  2. 또한 소프트웨어는 '시작' - '설정' - '제어판' - '프로그램 추가 또는 제거'를 통해 제거 가능합니다.  'GlobalTIS' 항목을 검색한 후 '변경/제거'를 클릭해야 합니다.

이전 설치 구성에 관한 일부 정보가 시스템에 그대로 남아 있게 됩니다.  예를 들어 등록 절차에 입력하고 받은 정보입니다.  이러한 정보는 다시 사용 가능하며 GlobalTIS를 다시 설치할 때 더욱 용이하게 해 줍니다.  하지만 GlobalTIS 설치 시, '예전 설정 무시하기'를 선택하시면 모든 등록 정보가 손실되어 라이센스 절차를 다시 실행해야 합니다.
